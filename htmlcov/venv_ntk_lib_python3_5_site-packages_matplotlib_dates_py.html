8Matplotlib date format 

9---------------------- 

10Matplotlib represents dates using floating point numbers specifying the number 

11of days since 0001-01-01 UTC, plus 1. For example, 0001-01-01, 06:00 is 1.25, 

12not 0.25. Values < 1, i.e. dates before 0001-01-01 UTC are not supported. 

13 

14There are a number of helper functions to convert between :mod:`datetime` 

15objects and Matplotlib dates:

31.. note:: 

32 

33 Like Python's datetime, mpl uses the Gregorian calendar for all 

34 conversions between dates and floating point numbers. This practice 

35 is not universal, and calendar differences can cause confusing 

36 differences between what Python and mpl give as the number of days 

37 since 0001-01-01 and what other software and databases yield. For 

38 example, the US Naval Observatory uses a calendar that switches 

39 from Julian to Gregorian in October, 1582. Hence, using their 

40 calculator, the number of days between 0001-01-01 and 2006-04-01 is 

41 732403, whereas using the Gregorian calendar via the datetime 

42 module we find:: 

43 

44 In [1]: date(2006, 4, 1).toordinal() - date(1, 1, 1).toordinal() 

45 Out[1]: 732401 

46 

47All the Matplotlib date converters, tickers and formatters are timezone aware. 

48If no explicit timezone is provided, the rcParam ``timezone`` is assumend. If 

49you want to use a custom time zone, pass a :class:`datetime.tzinfo` instance 

50with the tz keyword argument to :func:`num2date`, :func:`.plot_date`, and any 

51custom date tickers or locators you create. 

52 

53A wide range of specific and general purpose date tick locators and 

54formatters are provided in this module. See 

55:mod:`matplotlib.ticker` for general information on tick locators 

56and formatters. These are described below. 

57 

58 

59The `dateutil module <https://dateutil.readthedocs.io>`_ provides 

60additional code to handle date ticking, making it easy to place ticks 

61on any kinds of dates. See examples below.

63Date tickers 

64------------ 

65 

66Most of the date tickers can locate single or multiple values. For 

67example:: 

68 

69 # import constants for the days of the week 

70 from matplotlib.dates import MO, TU, WE, TH, FR, SA, SU 

71 

72 # tick on mondays every week 

73 loc = WeekdayLocator(byweekday=MO, tz=tz) 

74 

75 # tick on mondays and saturdays 

76 loc = WeekdayLocator(byweekday=(MO, SA)) 

77 

78In addition, most of the constructors take an interval argument:: 

79 

80 # tick on mondays every second week 

81 loc = WeekdayLocator(byweekday=MO, interval=2) 

82 

83The rrule locator allows completely general date ticking:: 

84 

85 # tick every 5th easter 

86 rule = rrulewrapper(YEARLY, byeaster=1, interval=5) 

87 loc = RRuleLocator(rule) 

88

90 

91 * :class:`MicrosecondLocator`: locate microseconds 

92 

93 * :class:`SecondLocator`: locate seconds 

94 

95 * :class:`MinuteLocator`: locate minutes 

96

98

100

118 locations. If called with ``interval_multiples=True`` it will

216def _to_ordinalf(dt): 

217 """ 

218 Convert :mod:`datetime` or :mod:`date` to the Gregorian date as UTC float 

219 days, preserving hours, minutes, seconds and microseconds. Return value 

220 is a :func:`float`. 

221 """ 

222 # Convert to UTC 

223 tzi = getattr(dt, 'tzinfo', None) 

224 if tzi is not None: 

225 dt = dt.astimezone(UTC) 

226 tzi = UTC 

227 

228 base = float(dt.toordinal()) 

229 

230 # If it's sufficiently datetime-like, it will have a `date()` method 

231 cdate = getattr(dt, 'date', lambda: None)() 

232 if cdate is not None: 

233 # Get a datetime object at midnight UTC 

234 midnight_time = datetime.time(0, tzinfo=tzi) 

235 

236 rdt = datetime.datetime.combine(cdate, midnight_time) 

237 

238 # Append the seconds as a fraction of a day 

239 base += (dt - rdt).total_seconds() / SEC_PER_DAY 

240 

241 return base

248def _dt64_to_ordinalf(d): 

249 """ 

250 Convert `numpy.datetime64` or an ndarray of those types to Gregorian 

251 date as UTC float. Roundoff is via float64 precision. Practically: 

252 microseconds for dates between 290301 BC, 294241 AD, milliseconds for 

253 larger dates (see `numpy.datetime64`). Nanoseconds aren't possible 

254 because we do times compared to ``0001-01-01T00:00:00`` (plus one day). 

255 """ 

256 

257 # the "extra" ensures that we at least allow the dynamic range out to 

258 # seconds. That should get out to +/-2e11 years. 

259 # NOTE: First cast truncates; second cast back is for NumPy 1.10. 

260 extra = d - d.astype('datetime64[s]').astype(d.dtype) 

261 extra = extra.astype('timedelta64[ns]') 

262 t0 = np.datetime64('0001-01-01T00:00:00').astype('datetime64[s]') 

263 dt = (d.astype('datetime64[s]') - t0).astype(np.float64) 

264 dt += extra.astype(np.float64) / 1.0e9 

265 dt = dt / SEC_PER_DAY + 1.0 

266 

267 NaT_int = np.datetime64('NaT').astype(np.int64) 

268 d_int = d.astype(np.int64) 

269 try: 

270 dt[d_int == NaT_int] = np.nan 

271 except TypeError: 

272 if d_int == NaT_int: 

273 dt = np.nan 

274 return dt

277def _from_ordinalf(x, tz=None): 

278 """ 

279 Convert Gregorian float of the date, preserving hours, minutes, 

280 seconds and microseconds. Return value is a `.datetime`. 

281 

282 The input date *x* is a float in ordinal days at UTC, and the output will 

283 be the specified `.datetime` object corresponding to that time in 

284 timezone *tz*, or if *tz* is ``None``, in the timezone specified in 

285 :rc:`timezone`. 

286 """ 

287 if tz is None: 

288 tz = _get_rc_timezone() 

289 

290 ix, remainder = divmod(x, 1) 

291 ix = int(ix) 

292 if ix < 1: 

293 raise ValueError('Cannot convert {} to a date. This often happens if ' 

294 'non-datetime values are passed to an axis that ' 

295 'expects datetime objects.'.format(ix)) 

296 dt = datetime.datetime.fromordinal(ix).replace(tzinfo=UTC) 

297 

298 # Since the input date `x` float is unable to preserve microsecond 

299 # precision of time representation in non-antique years, the 

300 # resulting datetime is rounded to the nearest multiple of 

301 # `musec_prec`. A value of 20 is appropriate for current dates. 

302 musec_prec = 20 

303 remainder_musec = int(round(remainder * MUSECONDS_PER_DAY / musec_prec) 

304 * musec_prec) 

305 

306 # For people trying to plot with full microsecond precision, enable 

307 # an early-year workaround 

308 if x < 30 * 365: 

309 remainder_musec = int(round(remainder * MUSECONDS_PER_DAY)) 

310 

311 # add hours, minutes, seconds, microseconds 

312 dt += datetime.timedelta(microseconds=remainder_musec) 

313 

314 return dt.astimezone(tz)

392def date2num(d): 

393 """ 

394 Convert datetime objects to Matplotlib dates. 

395 

396 Parameters 

397 ---------- 

398 d : `datetime.datetime` or `numpy.datetime64` or sequences of these 

399 

400 Returns 

401 ------- 

402 float or sequence of floats 

403 Number of days (fraction part represents hours, minutes, seconds, ms) 

404 since 0001-01-01 00:00:00 UTC, plus one. 

405 

406 Notes 

407 ----- 

408 The addition of one here is a historical artifact. Also, note that the 

409 Gregorian calendar is assumed; this is not universal practice. 

410 For details see the module docstring. 

411 """ 

412 if hasattr(d, "values"): 

413 # this unpacks pandas series or dataframes... 

414 d = d.values 

415 if not np.iterable(d): 

416 if (isinstance(d, np.datetime64) or (isinstance(d, np.ndarray) and 

417 np.issubdtype(d.dtype, np.datetime64))): 

418 return _dt64_to_ordinalf(d) 

419 return _to_ordinalf(d) 

420 

421 else: 

422 d = np.asarray(d) 

423 if np.issubdtype(d.dtype, np.datetime64): 

424 return _dt64_to_ordinalf(d) 

425 if not d.size: 

426 return d 

427 return _to_ordinalf_np_vectorized(d)

538def drange(dstart, dend, delta): 

539 """ 

540 Return a sequence of equally spaced Matplotlib dates. 

541 

542 The dates start at *dstart* and reach up to, but not including *dend*. 

543 They are spaced by *delta*. 

544 

545 Parameters 

546 ---------- 

547 dstart, dend : `~datetime.datetime` 

548 The date limits. 

549 delta : `datetime.timedelta` 

550 Spacing of the dates. 

551 

552 Returns 

553 ------- 

554 drange : `numpy.array` 

555 A list floats representing Matplotlib dates. 

556 

557 """ 

558 f1 = date2num(dstart) 

559 f2 = date2num(dend) 

560 step = delta.total_seconds() / SEC_PER_DAY 

561 

562 # calculate the difference between dend and dstart in times of delta 

563 num = int(np.ceil((f2 - f1) / step)) 

564 

565 # calculate end of the interval which will be generated 

566 dinterval_end = dstart + num * delta 

567 

568 # ensure, that an half open interval will be generated [dstart, dend) 

569 if dinterval_end >= dend: 

570 # if the endpoint is greated than dend, just subtract one delta 

571 dinterval_end -= delta 

572 num -= 1 

573 

574 f2 = date2num(dinterval_end) # new float-endpoint 

575 return np.linspace(f1, f2, num + 1)

614 @cbook.deprecated("3.0") 

615 def _replace_common_substr(self, s1, s2, sub1, sub2, replacement): 

616 """Helper function for replacing substrings sub1 and sub2 

617 located at the same indexes in strings s1 and s2 respectively, 

618 with the string replacement. It is expected that sub1 and sub2 

619 have the same length. Returns the pair s1, s2 after the 

620 substitutions. 

621 """ 

622 # Find common indexes of substrings sub1 in s1 and sub2 in s2 

623 # and make substitutions inplace. Because this is inplace, 

624 # it is okay if len(replacement) != len(sub1), len(sub2). 

625 i = 0 

626 while True: 

627 j = s1.find(sub1, i) 

628 if j == -1: 

629 break 

630 

631 i = j + 1 

632 if s2[j:j + len(sub2)] != sub2: 

633 continue 

634 

635 s1 = s1[:j] + replacement + s1[j + len(sub1):] 

636 s2 = s2[:j] + replacement + s2[j + len(sub2):] 

637 

638 return s1, s2

640 @cbook.deprecated("3.0") 

641 def strftime_pre_1900(self, dt, fmt=None): 

642 """Call time.strftime for years before 1900 by rolling 

643 forward a multiple of 28 years. 

644 

645 *fmt* is a :func:`strftime` format string. 

646 

647 Dalke: I hope I did this math right. Every 28 years the 

648 calendar repeats, except through century leap years excepting 

649 the 400 year leap years. But only if you're using the Gregorian 

650 calendar. 

651 """ 

652 if fmt is None: 

653 fmt = self.fmt 

654 

655 # Since python's time module's strftime implementation does not 

656 # support %f microsecond (but the datetime module does), use a 

657 # regular expression substitution to replace instances of %f. 

658 # Note that this can be useful since python's floating-point 

659 # precision representation for datetime causes precision to be 

660 # more accurate closer to year 0 (around the year 2000, precision 

661 # can be at 10s of microseconds). 

662 fmt = re.sub(r'((^|[^%])(%%)*)%f', 

663 r'\g<1>{0:06d}'.format(dt.microsecond), fmt) 

664 

665 year = dt.year 

666 # For every non-leap year century, advance by 

667 # 6 years to get into the 28-year repeat cycle 

668 delta = 2000 - year 

669 off = 6 * (delta // 100 + delta // 400) 

670 year = year + off 

671 

672 # Move to between the years 1973 and 2000 

673 year1 = year + ((2000 - year) // 28) * 28 

674 year2 = year1 + 28

677 s1 = time.strftime(fmt, (year1,) + timetuple[1:])

681 # that are located at the same indexes of s1, s2 with dt's year. 

682 # Note that C++'s strftime implementation does not use padded 

683 # zeros or padded whitespace for %y or %Y for years before 100, but 

684 # uses padded zeros for %x. (For example, try the runnable examples 

685 # with .tm_year in the interval [-1900, -1800] on 

686 # http://en.cppreference.com/w/c/chrono/strftime.) For ease of 

687 # implementation, we always use padded zeros for %y, %Y, and %x. 

688 s1, s2 = self._replace_common_substr(s1, s2, 

689 "{0:04d}".format(year1), 

690 "{0:04d}".format(year2), 

691 "{0:04d}".format(dt.year)) 

692 s1, s2 = self._replace_common_substr(s1, s2, 

693 "{0:02d}".format(year1 % 100), 

694 "{0:02d}".format(year2 % 100), 

695 "{0:02d}".format(dt.year % 100))

746class AutoDateFormatter(ticker.Formatter): 

747 """ 

748 This class attempts to figure out the best format to use. This is 

749 most useful when used with the :class:`AutoDateLocator`. 

750 

751 

752 The AutoDateFormatter has a scale dictionary that maps the scale 

753 of the tick (the distance in days between one major tick) and a 

754 format string. The default looks like this:: 

755 

756 self.scaled = { 

757 DAYS_PER_YEAR: rcParams['date.autoformat.year'], 

758 DAYS_PER_MONTH: rcParams['date.autoformat.month'], 

759 1.0: rcParams['date.autoformat.day'], 

760 1. / HOURS_PER_DAY: rcParams['date.autoformat.hour'], 

761 1. / (MINUTES_PER_DAY): rcParams['date.autoformat.minute'], 

762 1. / (SEC_PER_DAY): rcParams['date.autoformat.second'], 

763 1. / (MUSECONDS_PER_DAY): rcParams['date.autoformat.microsecond'], 

764 } 

765 

766 

767 The algorithm picks the key in the dictionary that is >= the 

768 current scale and uses that format string. You can customize this 

769 dictionary by doing:: 

770 

771 

772 >>> locator = AutoDateLocator() 

773 >>> formatter = AutoDateFormatter(locator) 

774 >>> formatter.scaled[1/(24.*60.)] = '%M:%S' # only show min and sec 

775 

776 A custom :class:`~matplotlib.ticker.FuncFormatter` can also be used. 

777 The following example shows how to use a custom format function to strip 

778 trailing zeros from decimal seconds and adds the date to the first 

779 ticklabel:: 

780 

781 >>> def my_format_function(x, pos=None): 

782 ... x = matplotlib.dates.num2date(x) 

783 ... if pos == 0: 

784 ... fmt = '%D %H:%M:%S.%f' 

785 ... else: 

786 ... fmt = '%H:%M:%S.%f' 

787 ... label = x.strftime(fmt) 

788 ... label = label.rstrip("0") 

789 ... label = label.rstrip(".") 

790 ... return label 

791 >>> from matplotlib.ticker import FuncFormatter 

792 >>> formatter.scaled[1/(24.*60.)] = FuncFormatter(my_format_function) 

793 """ 

794 

795 # This can be improved by providing some user-level direction on 

796 # how to choose the best format (precedence, etc...) 

797 

798 # Perhaps a 'struct' that has a field for each time-type where a 

799 # zero would indicate "don't show" and a number would indicate 

800 # "show" with some sort of priority. Same priorities could mean 

801 # show all with the same priority. 

802 

803 # Or more simply, perhaps just a format string for each 

804 # possibility... 

805 

806 def __init__(self, locator, tz=None, defaultfmt='%Y-%m-%d'): 

807 """ 

808 Autoformat the date labels. The default format is the one to use 

809 if none of the values in ``self.scaled`` are greater than the unit 

810 returned by ``locator._get_unit()``. 

811 """ 

812 self._locator = locator 

813 self._tz = tz 

814 self.defaultfmt = defaultfmt 

815 self._formatter = DateFormatter(self.defaultfmt, tz) 

816 self.scaled = {DAYS_PER_YEAR: rcParams['date.autoformatter.year'], 

817 DAYS_PER_MONTH: rcParams['date.autoformatter.month'], 

818 1.0: rcParams['date.autoformatter.day'], 

819 1. / HOURS_PER_DAY: rcParams['date.autoformatter.hour'], 

820 1. / (MINUTES_PER_DAY): 

821 rcParams['date.autoformatter.minute'], 

822 1. / (SEC_PER_DAY): 

823 rcParams['date.autoformatter.second'], 

824 1. / (MUSECONDS_PER_DAY): 

825 rcParams['date.autoformatter.microsecond']} 

826 

827 def __call__(self, x, pos=None): 

828 locator_unit_scale = float(self._locator._get_unit()) 

829 # Pick the first scale which is greater than the locator unit. 

830 fmt = next((fmt for scale, fmt in sorted(self.scaled.items()) 

831 if scale >= locator_unit_scale), 

832 self.defaultfmt) 

833 

834 if isinstance(fmt, str): 

835 self._formatter = DateFormatter(fmt, self._tz) 

836 result = self._formatter(x, pos) 

837 elif callable(fmt): 

838 result = fmt(x, pos) 

839 else: 

840 raise TypeError('Unexpected type passed to {0!r}.'.format(self)) 

841 

842 return result

857 def _update_rrule(self, **kwargs): 

858 tzinfo = self._base_tzinfo 

859 

860 # rrule does not play nicely with time zones - especially pytz time 

861 # zones, it's best to use naive zones and attach timezones once the 

862 # datetimes are returned 

863 if 'dtstart' in kwargs: 

864 dtstart = kwargs['dtstart'] 

865 if dtstart.tzinfo is not None: 

866 if tzinfo is None: 

867 tzinfo = dtstart.tzinfo 

868 else: 

869 dtstart = dtstart.astimezone(tzinfo)

871 kwargs['dtstart'] = dtstart.replace(tzinfo=None) 

872 

873 if 'until' in kwargs: 

874 until = kwargs['until'] 

875 if until.tzinfo is not None: 

876 if tzinfo is not None: 

877 until = until.astimezone(tzinfo) 

878 else: 

879 raise ValueError('until cannot be aware if dtstart ' 

880 'is naive and tzinfo is None') 

881 

882 kwargs['until'] = until.replace(tzinfo=None)

888 def _attach_tzinfo(self, dt, tzinfo):

891 return tzinfo.localize(dt, is_dst=True) 

892 

893 return dt.replace(tzinfo=tzinfo)

895 def _aware_return_wrapper(self, f, returns_list=False): 

896 """Decorator function that allows rrule methods to handle tzinfo.""" 

897 # This is only necessary if we're actually attaching a tzinfo 

898 if self._tzinfo is None: 

899 return f 

900 

901 # All datetime arguments must be naive. If they are not naive, they are 

902 # converted to the _tzinfo zone before dropping the zone. 

903 def normalize_arg(arg): 

904 if isinstance(arg, datetime.datetime) and arg.tzinfo is not None: 

905 if arg.tzinfo is not self._tzinfo: 

906 arg = arg.astimezone(self._tzinfo) 

907 

908 return arg.replace(tzinfo=None) 

909 

910 return arg

912 def normalize_args(args, kwargs): 

913 args = tuple(normalize_arg(arg) for arg in args) 

914 kwargs = {kw: normalize_arg(arg) for kw, arg in kwargs.items()} 

915 

916 return args, kwargs 

917 

918 # There are two kinds of functions we care about - ones that return 

919 # dates and ones that return lists of dates. 

920 if not returns_list: 

921 def inner_func(*args, **kwargs): 

922 args, kwargs = normalize_args(args, kwargs) 

923 dt = f(*args, **kwargs) 

924 return self._attach_tzinfo(dt, self._tzinfo) 

925 else: 

926 def inner_func(*args, **kwargs): 

927 args, kwargs = normalize_args(args, kwargs) 

928 dts = f(*args, **kwargs) 

929 return [self._attach_tzinfo(dt, self._tzinfo) for dt in dts] 

930 

931 return functools.wraps(f)(inner_func)

938

940 return self._aware_return_wrapper(f) 

941 elif name in {'xafter', 'xbefore', 'between'}:

1277 # whenever possible. 

1278 numYears = float(delta.years) 

1279 numMonths = numYears * MONTHS_PER_YEAR + delta.months 

1280 numDays = tdelta.days # Avoids estimates of days/month, days/year 

1281 numHours = numDays * HOURS_PER_DAY + delta.hours 

1282 numMinutes = numHours * MIN_PER_HOUR + delta.minutes 

1283 numSeconds = np.floor(tdelta.total_seconds()) 

1284 numMicroseconds = np.floor(tdelta.total_seconds() * 1e6) 

1285 

1286 nums = [numYears, numMonths, numDays, numHours, numMinutes, 

1287 numSeconds, numMicroseconds] 

1288 

1289 use_rrule_locator = [True] * 6 + [False] 

1290 

1291 # Default setting of bymonth, etc. to pass to rrule 

1292 # [unused (for year), bymonth, bymonthday, byhour, byminute, 

1293 # bysecond, unused (for microseconds)] 

1294 byranges = [None, 1, 1, 0, 0, 0, None] 

1295 

1296 # Loop over all the frequencies and try to find one that gives at 

1297 # least a minticks tick positions. Once this is found, look for 

1298 # an interval from an list specific to that frequency that gives no 

1299 # more than maxticks tick positions. Also, set up some ranges 

1300 # (bymonth, etc.) as appropriate to be passed to rrulewrapper. 

1301 for i, (freq, num) in enumerate(zip(self._freqs, nums)): 

1302 # If this particular frequency doesn't give enough ticks, continue 

1303 if num < self.minticks: 

1304 # Since we're not using this particular frequency, set 

1305 # the corresponding by_ to None so the rrule can act as 

1306 # appropriate 

1307 byranges[i] = None 

1308 continue 

1309 

1310 # Find the first available interval that doesn't give too many 

1311 # ticks 

1312 for interval in self.intervald[freq]: 

1313 if num <= interval * (self.maxticks[freq] - 1): 

1314 break 

1315 else: 

1316 # We went through the whole loop without breaking, default to 

1317 # the last interval in the list and raise a warning 

1318 warnings.warn('AutoDateLocator was unable to pick an ' 

1319 'appropriate interval for this date range. ' 

1320 'It may be necessary to add an interval value ' 

1321 "to the AutoDateLocator's intervald dictionary." 

1322 ' Defaulting to {0}.'.format(interval)) 

1323 

1324 # Set some parameters as appropriate 

1325 self._freq = freq 

1326 

1327 if self._byranges[i] and self.interval_multiples: 

1328 if i == DAILY and interval == 14: 

1329 # just make first and 15th. Avoids 30th. 

1330 byranges[i] = [1, 15] 

1331 else: 

1332 byranges[i] = self._byranges[i][::interval] 

1333 interval = 1 

1334 else: 

1335 byranges[i] = self._byranges[i] 

1336 

1337 break 

1338 else: 

1339 raise ValueError('No sensible date limit could be found in the ' 

1340 'AutoDateLocator.') 

1341 

1342 if (freq == YEARLY) and self.interval_multiples: 

1343 locator = YearLocator(interval, tz=self.tz) 

1344 elif use_rrule_locator[i]: 

1345 _, bymonth, bymonthday, byhour, byminute, bysecond, _ = byranges 

1346 rrule = rrulewrapper(self._freq, interval=interval, 

1347 dtstart=dmin, until=dmax, 

1348 bymonth=bymonth, bymonthday=bymonthday, 

1349 byhour=byhour, byminute=byminute, 

1350 bysecond=bysecond) 

1351 

1352 locator = RRuleLocator(rrule, self.tz) 

1353 else: 

1354 locator = MicrosecondLocator(interval, tz=self.tz) 

1355 if dmin.year > 20 and interval < 1000: 

1356 _log.warn('Plotting microsecond time intervals is not' 

1357 ' well supported. Please see the' 

1358 ' MicrosecondLocator documentation' 

1359 ' for details.') 

1360 

1361 locator.set_axis(self.axis) 

1362 

1363 if self.axis is not None: 

1364 locator.set_view_interval(*self.axis.get_view_interval()) 

1365 locator.set_data_interval(*self.axis.get_data_interval()) 

1366 return locator
