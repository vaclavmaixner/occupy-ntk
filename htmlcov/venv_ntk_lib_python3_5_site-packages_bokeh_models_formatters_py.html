109class MercatorTickFormatter(BasicTickFormatter): 

110 ''' A ``TickFormatter`` for values in WebMercator units. 

111 

112 Some map plot types internally use WebMercator to describe coordinates, 

113 plot bounds, etc. These units are not very human-friendly. This tick 

114 formatter will convert WebMercator units into Latitude and Longitude 

115 for display on axes. 

116 

117 ''' 

118 

119 dimension = Enum(LatLon, default=None, help=""" 

120 Specify whether to format ticks for Latitude or Longitude. 

121 

122 Projected coordinates are not separable, computing Latitude and Longitude 

123 tick labels from Web Mercator requires considering coordinates from both 

124 dimensions together. Use this property to specify which result should be 

125 used for display. 

126 

127 Typically, if the formatter is for an x-axis, then dimension should be 

128 ``"lon"`` and if the formatter is for a y-axis, then the dimension 

129 should be `"lat"``. 

130 

131 In order to prevent hard to debug errors, there is no default value for 

132 dimension. Using an un-configured ``MercatorTickFormatter`` will result in 

133 a validation error and a JavaScript console error. 

134 """)

306 def from_py_func(cls, func): 

307 ''' Create a ``FuncTickFormatter`` instance from a Python function. The 

308 function is translated to JavaScript using PScript. The variable 

309 ``tick`` will contain the unformatted tick value and can be expected to 

310 be present in the function namespace at render time. 

311 

312 Example: 

313 

314 .. code-block:: python 

315 

316 code = """ 

317 def ticker(): 

318 return "{:.0f} + {:.2f}".format(tick, tick % 1) 

319 """ 

320 

321 The python function must have no positional arguments. It's 

322 possible to pass Bokeh models (e.g. a ``ColumnDataSource``) as keyword 

323 arguments to the function. 

324 

325 ''' 

326 from bokeh.util.deprecation import deprecated 

327 deprecated("'from_py_func' is deprecated and will be removed in an eventual 2.0 release. " 

328 "Use FuncTickFormatter directly instead.") 

329 

330 if not isinstance(func, FunctionType): 

331 raise ValueError('CustomJS.from_py_func needs function object.') 

332 pscript = import_required('pscript', 

333 'To use Python functions for CustomJS, you need PScript ' + 

334 '("conda install -c conda-forge pscript" or "pip install pscript")') 

335 sig = signature(func) 

336 

337 all_names, default_values = get_param_info(sig) 

338 

339 if len(all_names) - len(default_values) != 0: 

340 raise ValueError("Function `func` may only contain keyword arguments.") 

341 

342 if default_values and not any(isinstance(value, Model) for value in default_values): 

343 raise ValueError("Default value must be a Bokeh Model.") 

344 

345 func_kwargs = dict(zip(all_names, default_values)) 

346 

347 # Wrap the code attr in a function named `formatter` and call it 

348 # with arguments that match the `args` attr 

349 code = pscript.py2js(func, 'formatter') + 'return formatter(%s);\n' % ', '.join(all_names) 

350 

351 return cls(code=code, args=func_kwargs)

408class DatetimeTickFormatter(TickFormatter): 

409 ''' A ``TickFormatter`` for displaying datetime values nicely across a 

410 range of scales. 

411 

412 ``DatetimeTickFormatter`` has the following properties (listed together 

413 with their default values) that can be used to control the formatting 

414 of axis ticks at different scales scales: 

415 

416 .. code-block:: python 

417 

418 {defaults} 

419 

420 Each scale property can be set to format or list of formats to use for 

421 formatting datetime tick values that fall in in that "time scale". 

422 By default, only the first format string passed for each time scale 

423 will be used. By default, all leading zeros are stripped away from 

424 the formatted labels. 

425 

426 This list of supported `strftime`_ formats is reproduced below. 

427 

428 %a 

429 The abbreviated name of the day of the week according to the 

430 current locale. 

431 

432 %A 

433 The full name of the day of the week according to the current 

434 locale. 

435 

436 %b 

437 The abbreviated month name according to the current locale. 

438 

439 %B 

440 The full month name according to the current locale. 

441 

442 %c 

443 The preferred date and time representation for the current 

444 locale. 

445 

446 %C 

447 The century number (year/100) as a 2-digit integer. 

448 

449 %d 

450 The day of the month as a decimal number (range 01 to 31). 

451 

452 %D 

453 Equivalent to %m/%d/%y. (Americans should note that in many 

454 other countries %d/%m/%y is rather common. This means that in 

455 international context this format is ambiguous and should not 

456 be used.) 

457 

458 %e 

459 Like %d, the day of the month as a decimal number, but a 

460 leading zero is replaced by a space. 

461 

462 %f 

463 Microsecond as a decimal number, zero-padded on the left (range 

464 000000-999999). This is an extension to the set of directives 

465 available to `timezone`_. 

466 

467 %F 

468 Equivalent to %Y-%m-%d (the ISO 8601 date format). 

469 

470 %G 

471 The ISO 8601 week-based year with century as a decimal number. 

472 The 4-digit year corresponding to the ISO week number (see %V). 

473 This has the same format and value as %Y, except that if the 

474 ISO week number belongs to the previous or next year, that year 

475 is used instead. 

476 

477 %g 

478 Like %G, but without century, that is, with a 2-digit year (00-99). 

479 

480 %h 

481 Equivalent to %b. 

482 

483 %H 

484 The hour as a decimal number using a 24-hour clock (range 00 

485 to 23). 

486 

487 %I 

488 The hour as a decimal number using a 12-hour clock (range 01 

489 to 12). 

490 

491 %j 

492 The day of the year as a decimal number (range 001 to 366). 

493 

494 %k 

495 The hour (24-hour clock) as a decimal number (range 0 to 23). 

496 Single digits are preceded by a blank. (See also %H.) 

497 

498 %l 

499 The hour (12-hour clock) as a decimal number (range 1 to 12). 

500 Single digits are preceded by a blank. (See also %I.) (TZ) 

501 

502 %m 

503 The month as a decimal number (range 01 to 12). 

504 

505 %M 

506 The minute as a decimal number (range 00 to 59). 

507 

508 %n 

509 A newline character. Bokeh text does not currently support 

510 newline characters. 

511 

512 %N 

513 Nanosecond as a decimal number, zero-padded on the left (range 

514 000000000-999999999). Supports a padding width specifier, i.e. 

515 %3N displays 3 leftmost digits. However, this is only accurate 

516 to the millisecond level of precision due to limitations of 

517 `timezone`_. 

518 

519 %p 

520 Either "AM" or "PM" according to the given time value, or the 

521 corresponding strings for the current locale. Noon is treated 

522 as "PM" and midnight as "AM". 

523 

524 %P 

525 Like %p but in lowercase: "am" or "pm" or a corresponding 

526 string for the current locale. 

527 

528 %r 

529 The time in a.m. or p.m. notation. In the POSIX locale this 

530 is equivalent to %I:%M:%S %p. 

531 

532 %R 

533 The time in 24-hour notation (%H:%M). For a version including 

534 the seconds, see %T below. 

535 

536 %s 

537 The number of seconds since the Epoch, 1970-01-01 00:00:00 

538 +0000 (UTC). 

539 

540 %S 

541 The second as a decimal number (range 00 to 60). (The range 

542 is up to 60 to allow for occasional leap seconds.) 

543 

544 %t 

545 A tab character. Bokeh text does not currently support tab 

546 characters. 

547 

548 %T 

549 The time in 24-hour notation (%H:%M:%S). 

550 

551 %u 

552 The day of the week as a decimal, range 1 to 7, Monday being 1. 

553 See also %w. 

554 

555 %U 

556 The week number of the current year as a decimal number, range 

557 00 to 53, starting with the first Sunday as the first day of 

558 week 01. See also %V and %W. 

559 

560 %V 

561 The ISO 8601 week number (see NOTES) of the current year as a 

562 decimal number, range 01 to 53, where week 1 is the first week 

563 that has at least 4 days in the new year. See also %U and %W. 

564 

565 %w 

566 The day of the week as a decimal, range 0 to 6, Sunday being 0. 

567 See also %u. 

568 

569 %W 

570 The week number of the current year as a decimal number, range 

571 00 to 53, starting with the first Monday as the first day of 

572 week 01. 

573 

574 %x 

575 The preferred date representation for the current locale 

576 without the time. 

577 

578 %X 

579 The preferred time representation for the current locale 

580 without the date. 

581 

582 %y 

583 The year as a decimal number without a century (range 00 to 99). 

584 

585 %Y 

586 The year as a decimal number including the century. 

587 

588 %z 

589 The +hhmm or -hhmm numeric timezone (that is, the hour and 

590 minute offset from UTC). 

591 

592 %Z 

593 The timezone name or abbreviation. 

594 

595 %% 

596 A literal '%' character. 

597 

598 .. warning:: 

599 The client library BokehJS uses the `timezone`_ library to 

600 format datetimes. The inclusion of the list below is based on the 

601 claim that `timezone`_ makes to support "the full compliment 

602 of GNU date format specifiers." However, this claim has not 

603 been tested exhaustively against this list. If you find formats 

604 that do not function as expected, please submit a `github issue`_, 

605 so that the documentation can be updated appropriately. 

606 

607 .. _strftime: http://man7.org/linux/man-pages/man3/strftime.3.html 

608 .. _timezone: http://bigeasy.github.io/timezone/ 

609 .. _github issue: https://github.com/bokeh/bokeh/issues 

610
