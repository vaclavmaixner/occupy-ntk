1303 :meth:`~.RelationshipProperty.Comparator.contains` is

1305 operations.

1308 :meth:`~.RelationshipProperty.Comparator.contains`

1325 :meth:`~.RelationshipProperty.Comparator.contains`

1327 used in queries that move beyond simple AND 

1328 conjunctions, such as multiple 

1329 :meth:`~.RelationshipProperty.Comparator.contains` 

1330 expressions joined by OR. In such cases subqueries or 

1331 explicit "outer joins" will need to be used instead. 

1332 See :meth:`~.RelationshipProperty.Comparator.any` for 

1333 a less-performant alternative using EXISTS, or refer 

1334 to :meth:`.Query.outerjoin` as well as :ref:`ormtutorial_joins` 

1335 for more details on constructing outer joins.

1533 def _get_attr_w_warn_on_none(self, mapper, state, dict_, column): 

1534 """Create the callable that is used in a many-to-one expression. 

1535 

1536 E.g.:: 

1537 

1538 u1 = s.query(User).get(5) 

1539 

1540 expr = Address.user == u1 

1541

1543 returned by this method produces the value "5" based on the identity 

1544 of ``u1``. 

1545 

1546 """ 

1547 

1548 # in this callable, we're trying to thread the needle through 

1549 # a wide variety of scenarios, including: 

1550 # 

1551 # * the object hasn't been flushed yet and there's no value for 

1552 # the attribute as of yet 

1553 # 

1554 # * the object hasn't been flushed yet but it has a user-defined 

1555 # value 

1556 # 

1557 # * the object has a value but it's expired and not locally present 

1558 # 

1559 # * the object has a value but it's expired and not locally present, 

1560 # and the object is also detached 

1561 # 

1562 # * The object hadn't been flushed yet, there was no value, but 

1563 # later, the object has been expired and detached, and *now* 

1564 # they're trying to evaluate it 

1565 # 

1566 # * the object had a value, but it was changed to a new value, and 

1567 # then expired 

1568 # 

1569 # * the object had a value, but it was changed to a new value, and 

1570 # then expired, then the object was detached 

1571 # 

1572 # * the object has a user-set value, but it's None and we don't do 

1573 # the comparison correctly for that so warn 

1574 # 

1575 

1576 prop = mapper.get_property_by_column(column) 

1577 

1578 # by invoking this method, InstanceState will track the last known 

1579 # value for this key each time the attribute is to be expired. 

1580 # this feature was added explicitly for use in this method. 

1581 state._track_last_known_value(prop.key) 

1582 

1583 def _go(): 

1584 last_known = to_return = state._last_known_values[prop.key] 

1585 existing_is_available = last_known is not attributes.NO_VALUE 

1586 

1587 # we support that the value may have changed. so here we 

1588 # try to get the most recent value including re-fetching. 

1589 # only if we can't get a value now due to detachment do we return 

1590 # the last known value 

1591 current_value = mapper._get_state_attr_by_column( 

1592 state, 

1593 dict_, 

1594 column, 

1595 passive=attributes.PASSIVE_OFF 

1596 if state.persistent 

1597 else attributes.PASSIVE_NO_FETCH ^ attributes.INIT_OK, 

1598 ) 

1599 

1600 if current_value is attributes.NEVER_SET: 

1601 if not existing_is_available: 

1602 raise sa_exc.InvalidRequestError( 

1603 "Can't resolve value for column %s on object " 

1604 "%s; no value has been set for this column" 

1605 % (column, state_str(state)) 

1606 ) 

1607 elif current_value is attributes.PASSIVE_NO_RESULT: 

1608 if not existing_is_available: 

1609 raise sa_exc.InvalidRequestError(

1613 )

1615 to_return = current_value 

1616 if to_return is None: 

1617 util.warn( 

1618 "Got None for value of column %s; this is unsupported "

1620 "currently produce an IS comparison " 

1621 "(but may in a future release)" % column
