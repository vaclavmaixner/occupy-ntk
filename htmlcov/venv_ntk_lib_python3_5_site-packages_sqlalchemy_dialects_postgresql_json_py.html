102class JSON(sqltypes.JSON): 

103 """Represent the PostgreSQL JSON type. 

104 

105 This type is a specialization of the Core-level :class:`.types.JSON` 

106 type. Be sure to read the documentation for :class:`.types.JSON` for 

107 important tips regarding treatment of NULL values and ORM use. 

108 

109 .. versionchanged:: 1.1 :class:`.postgresql.JSON` is now a PostgreSQL- 

110 specific specialization of the new :class:`.types.JSON` type. 

111 

112 The operators provided by the PostgreSQL version of :class:`.JSON` 

113 include: 

114 

115 * Index operations (the ``->`` operator):: 

116 

117 data_table.c.data['some key'] 

118 

119 data_table.c.data[5] 

120 

121 

122 * Index operations returning text (the ``->>`` operator):: 

123 

124 data_table.c.data['some key'].astext == 'some value' 

125 

126 Note that equivalent functionality is available via the 

127 :attr:`.JSON.Comparator.as_string` accessor. 

128 

129 * Index operations with CAST 

130 (equivalent to ``CAST(col ->> ['some key'] AS <type>)``):: 

131 

132 data_table.c.data['some key'].astext.cast(Integer) == 5 

133 

134 Note that equivalent functionality is available via the 

135 :attr:`.JSON.Comparator.as_integer` and similar accessors. 

136 

137 * Path index operations (the ``#>`` operator):: 

138 

139 data_table.c.data[('key_1', 'key_2', 5, ..., 'key_n')] 

140 

141 * Path index operations returning text (the ``#>>`` operator):: 

142 

143 data_table.c.data[('key_1', 'key_2', 5, ..., 'key_n')].astext == 'some value' 

144 

145 .. versionchanged:: 1.1 The :meth:`.ColumnElement.cast` operator on 

146 JSON objects now requires that the :attr:`.JSON.Comparator.astext` 

147 modifier be called explicitly, if the cast works only from a textual 

148 string. 

149 

150 Index operations return an expression object whose type defaults to 

151 :class:`.JSON` by default, so that further JSON-oriented instructions 

152 may be called upon the result type. 

153 

154 Custom serializers and deserializers are specified at the dialect level, 

155 that is using :func:`.create_engine`. The reason for this is that when 

156 using psycopg2, the DBAPI only allows serializers at the per-cursor 

157 or per-connection level. E.g.:: 

158 

159 engine = create_engine("postgresql://scott:tiger@localhost/test", 

160 json_serializer=my_serialize_fn, 

161 json_deserializer=my_deserialize_fn 

162 ) 

163 

164 When using the psycopg2 dialect, the json_deserializer is registered 

165 against the database using ``psycopg2.extras.register_default_json``. 

166 

167 .. seealso:: 

168 

169 :class:`.types.JSON` - Core level JSON type 

170 

171 :class:`.JSONB` 

172 

173 """ # noqa

241class JSONB(JSON): 

242 """Represent the PostgreSQL JSONB type. 

243 

244 The :class:`.JSONB` type stores arbitrary JSONB format data, e.g.:: 

245 

246 data_table = Table('data_table', metadata, 

247 Column('id', Integer, primary_key=True), 

248 Column('data', JSONB) 

249 ) 

250 

251 with engine.connect() as conn: 

252 conn.execute( 

253 data_table.insert(), 

254 data = {"key1": "value1", "key2": "value2"} 

255 ) 

256 

257 The :class:`.JSONB` type includes all operations provided by 

258 :class:`.JSON`, including the same behaviors for indexing operations. 

259 It also adds additional operators specific to JSONB, including 

260 :meth:`.JSONB.Comparator.has_key`, :meth:`.JSONB.Comparator.has_all`, 

261 :meth:`.JSONB.Comparator.has_any`, :meth:`.JSONB.Comparator.contains`, 

262 and :meth:`.JSONB.Comparator.contained_by`. 

263 

264 Like the :class:`.JSON` type, the :class:`.JSONB` type does not detect 

265 in-place changes when used with the ORM, unless the 

266 :mod:`sqlalchemy.ext.mutable` extension is used. 

267 

268 Custom serializers and deserializers 

269 are shared with the :class:`.JSON` class, using the ``json_serializer`` 

270 and ``json_deserializer`` keyword arguments. These must be specified 

271 at the dialect level using :func:`.create_engine`. When using 

272 psycopg2, the serializers are associated with the jsonb type using 

273 ``psycopg2.extras.register_default_jsonb`` on a per-connection basis, 

274 in the same way that ``psycopg2.extras.register_default_json`` is used 

275 to register these handlers with the json type. 

276 

277 .. versionadded:: 0.9.7 

278 

279 .. seealso:: 

280 

281 :class:`.JSON` 

282 

283 """ 

284
