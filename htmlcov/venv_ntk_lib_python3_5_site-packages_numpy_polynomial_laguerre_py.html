97 

98 Returns

128def lag2poly(c): 

129 """ 

130 Convert a Laguerre series to a polynomial. 

131 

132 Convert an array representing the coefficients of a Laguerre series, 

133 ordered from lowest degree to highest, to an array of the coefficients 

134 of the equivalent polynomial (relative to the "standard" basis) ordered 

135 from lowest to highest degree. 

136 

137 Parameters 

138 ---------- 

139 c : array_like 

140 1-D array containing the Laguerre series coefficients, ordered 

141 from lowest order term to highest. 

142 

143 Returns 

144 ------- 

145 pol : ndarray 

146 1-D array containing the coefficients of the equivalent polynomial 

147 (relative to the "standard" basis) ordered from lowest order term 

148 to highest. 

149 

150 See Also 

151 -------- 

152 poly2lag 

153 

154 Notes 

155 ----- 

156 The easy way to do conversions between polynomial basis sets 

157 is to use the convert method of a class instance. 

158 

159 Examples 

160 -------- 

161 >>> from numpy.polynomial.laguerre import lag2poly 

162 >>> lag2poly([ 23., -63., 58., -18.]) 

163 array([0., 1., 2., 3.]) 

164 

165 """ 

166 from .polynomial import polyadd, polysub, polymulx 

167 

168 [c] = pu.as_series([c]) 

169 n = len(c) 

170 if n == 1: 

171 return c 

172 else: 

173 c0 = c[-2] 

174 c1 = c[-1] 

175 # i is the current degree of c1 

176 for i in range(n - 1, 1, -1): 

177 tmp = c0 

178 c0 = polysub(c[i - 2], (c1*(i - 1))/i) 

179 c1 = polyadd(tmp, polysub((2*i - 1)*c1, polymulx(c1))/i) 

180 return polyadd(c0, polysub(c1, polymulx(c1)))

236def lagfromroots(roots): 

237 """ 

238 Generate a Laguerre series with given roots. 

239 

240 The function returns the coefficients of the polynomial 

241 

242 .. math:: p(x) = (x - r_0) * (x - r_1) * ... * (x - r_n), 

243 

244 in Laguerre form, where the `r_n` are the roots specified in `roots`. 

245 If a zero has multiplicity n, then it must appear in `roots` n times. 

246 For instance, if 2 is a root of multiplicity three and 3 is a root of 

247 multiplicity 2, then `roots` looks something like [2, 2, 2, 3, 3]. The 

248 roots can appear in any order. 

249 

250 If the returned coefficients are `c`, then 

251 

252 .. math:: p(x) = c_0 + c_1 * L_1(x) + ... + c_n * L_n(x) 

253 

254 The coefficient of the last term is not generally 1 for monic 

255 polynomials in Laguerre form. 

256 

257 Parameters 

258 ---------- 

259 roots : array_like 

260 Sequence containing the roots. 

261 

262 Returns 

263 ------- 

264 out : ndarray 

265 1-D array of coefficients. If all roots are real then `out` is a 

266 real array, if some of the roots are complex, then `out` is complex 

267 even if all the coefficients in the result are real (see Examples 

268 below). 

269 

270 See Also 

271 -------- 

272 polyfromroots, legfromroots, chebfromroots, hermfromroots, hermefromroots 

273 

274 Examples 

275 -------- 

276 >>> from numpy.polynomial.laguerre import lagfromroots, lagval 

277 >>> coef = lagfromroots((-1, 0, 1)) 

278 >>> lagval((-1, 0, 1), coef) 

279 array([0., 0., 0.]) 

280 >>> coef = lagfromroots((-1j, 1j)) 

281 >>> lagval((-1j, 1j), coef) 

282 array([0.+0.j, 0.+0.j]) 

283 

284 """ 

285 return pu._fromroots(lagline, lagmul, roots)

369def lagmulx(c): 

370 """Multiply a Laguerre series by x. 

371 

372 Multiply the Laguerre series `c` by x, where x is the independent 

373 variable. 

374 

375 

376 Parameters 

377 ---------- 

378 c : array_like 

379 1-D array of Laguerre series coefficients ordered from low to 

380 high. 

381 

382 Returns 

383 ------- 

384 out : ndarray 

385 Array representing the result of the multiplication. 

386 

387 See Also 

388 -------- 

389 lagadd, lagsub, lagmul, lagdiv, lagpow 

390 

391 Notes 

392 ----- 

393 The multiplication uses the recursion relationship for Laguerre 

394 polynomials in the form 

395 

396 .. math:: 

397 

398 xP_i(x) = (-(i + 1)*P_{i + 1}(x) + (2i + 1)P_{i}(x) - iP_{i - 1}(x)) 

399 

400 Examples 

401 -------- 

402 >>> from numpy.polynomial.laguerre import lagmulx 

403 >>> lagmulx([1, 2, 3]) 

404 array([-1., -1., 11., -9.]) 

405 

406 """ 

407 # c is a trimmed copy 

408 [c] = pu.as_series([c]) 

409 # The zero series needs special treatment 

410 if len(c) == 1 and c[0] == 0: 

411 return c 

412 

413 prd = np.empty(len(c) + 1, dtype=c.dtype) 

414 prd[0] = c[0] 

415 prd[1] = -c[0] 

416 for i in range(1, len(c)): 

417 prd[i + 1] = -c[i]*(i + 1) 

418 prd[i] += c[i]*(2*i + 1) 

419 prd[i - 1] -= c[i]*i 

420 return prd

423def lagmul(c1, c2): 

424 """ 

425 Multiply one Laguerre series by another. 

426 

427 Returns the product of two Laguerre series `c1` * `c2`. The arguments 

428 are sequences of coefficients, from lowest order "term" to highest, 

429 e.g., [1,2,3] represents the series ``P_0 + 2*P_1 + 3*P_2``. 

430 

431 Parameters 

432 ---------- 

433 c1, c2 : array_like 

434 1-D arrays of Laguerre series coefficients ordered from low to 

435 high. 

436 

437 Returns 

438 ------- 

439 out : ndarray 

440 Of Laguerre series coefficients representing their product. 

441 

442 See Also 

443 -------- 

444 lagadd, lagsub, lagmulx, lagdiv, lagpow 

445 

446 Notes 

447 ----- 

448 In general, the (polynomial) product of two C-series results in terms 

449 that are not in the Laguerre polynomial basis set. Thus, to express 

450 the product as a Laguerre series, it is necessary to "reproject" the 

451 product onto said basis set, which may produce "unintuitive" (but 

452 correct) results; see Examples section below. 

453 

454 Examples 

455 -------- 

456 >>> from numpy.polynomial.laguerre import lagmul 

457 >>> lagmul([1, 2, 3], [0, 1, 2]) 

458 array([ 8., -13., 38., -51., 36.]) 

459 

460 """ 

461 # s1, s2 are trimmed copies 

462 [c1, c2] = pu.as_series([c1, c2]) 

463 

464 if len(c1) > len(c2): 

465 c = c2 

466 xs = c1 

467 else: 

468 c = c1 

469 xs = c2 

470 

471 if len(c) == 1: 

472 c0 = c[0]*xs 

473 c1 = 0 

474 elif len(c) == 2: 

475 c0 = c[0]*xs 

476 c1 = c[1]*xs 

477 else: 

478 nd = len(c) 

479 c0 = c[-2]*xs 

480 c1 = c[-1]*xs 

481 for i in range(3, len(c) + 1): 

482 tmp = c0 

483 nd = nd - 1 

484 c0 = lagsub(c[-i]*xs, (c1*(nd - 1))/nd) 

485 c1 = lagadd(tmp, lagsub((2*nd - 1)*c1, lagmulx(c1))/nd) 

486 return lagadd(c0, lagsub(c1, lagmulx(c1)))

572def lagder(c, m=1, scl=1, axis=0): 

573 """ 

574 Differentiate a Laguerre series. 

575 

576 Returns the Laguerre series coefficients `c` differentiated `m` times 

577 along `axis`. At each iteration the result is multiplied by `scl` (the 

578 scaling factor is for use in a linear change of variable). The argument 

579 `c` is an array of coefficients from low to high degree along each 

580 axis, e.g., [1,2,3] represents the series ``1*L_0 + 2*L_1 + 3*L_2`` 

581 while [[1,2],[1,2]] represents ``1*L_0(x)*L_0(y) + 1*L_1(x)*L_0(y) + 

582 2*L_0(x)*L_1(y) + 2*L_1(x)*L_1(y)`` if axis=0 is ``x`` and axis=1 is 

583 ``y``. 

584 

585 Parameters 

586 ---------- 

587 c : array_like 

588 Array of Laguerre series coefficients. If `c` is multidimensional 

589 the different axis correspond to different variables with the 

590 degree in each axis given by the corresponding index. 

591 m : int, optional 

592 Number of derivatives taken, must be non-negative. (Default: 1) 

593 scl : scalar, optional 

594 Each differentiation is multiplied by `scl`. The end result is 

595 multiplication by ``scl**m``. This is for use in a linear change of 

596 variable. (Default: 1) 

597 axis : int, optional 

598 Axis over which the derivative is taken. (Default: 0). 

599 

600 .. versionadded:: 1.7.0 

601 

602 Returns 

603 ------- 

604 der : ndarray 

605 Laguerre series of the derivative. 

606 

607 See Also 

608 -------- 

609 lagint 

610 

611 Notes 

612 ----- 

613 In general, the result of differentiating a Laguerre series does not 

614 resemble the same operation on a power series. Thus the result of this 

615 function may be "unintuitive," albeit correct; see Examples section 

616 below. 

617 

618 Examples 

619 -------- 

620 >>> from numpy.polynomial.laguerre import lagder 

621 >>> lagder([ 1., 1., 1., -3.]) 

622 array([1., 2., 3.]) 

623 >>> lagder([ 1., 0., 0., -4., 3.], m=2) 

624 array([1., 2., 3.]) 

625 

626 """ 

627 c = np.array(c, ndmin=1, copy=True) 

628 if c.dtype.char in '?bBhHiIlLqQpP': 

629 c = c.astype(np.double) 

630 

631 cnt = pu._deprecate_as_int(m, "the order of derivation") 

632 iaxis = pu._deprecate_as_int(axis, "the axis") 

633 if cnt < 0: 

634 raise ValueError("The order of derivation must be non-negative") 

635 iaxis = normalize_axis_index(iaxis, c.ndim) 

636 

637 if cnt == 0: 

638 return c 

639 

640 c = np.moveaxis(c, iaxis, 0) 

641 n = len(c) 

642 if cnt >= n: 

643 c = c[:1]*0 

644 else: 

645 for i in range(cnt): 

646 n = n - 1 

647 c *= scl 

648 der = np.empty((n,) + c.shape[1:], dtype=c.dtype) 

649 for j in range(n, 1, -1): 

650 der[j - 1] = -c[j] 

651 c[j - 1] += c[j] 

652 der[0] = -c[1] 

653 c = der 

654 c = np.moveaxis(c, 0, iaxis) 

655 return c

658def lagint(c, m=1, k=[], lbnd=0, scl=1, axis=0): 

659 """ 

660 Integrate a Laguerre series. 

661 

662 Returns the Laguerre series coefficients `c` integrated `m` times from 

663 `lbnd` along `axis`. At each iteration the resulting series is 

664 **multiplied** by `scl` and an integration constant, `k`, is added. 

665 The scaling factor is for use in a linear change of variable. ("Buyer 

666 beware": note that, depending on what one is doing, one may want `scl` 

667 to be the reciprocal of what one might expect; for more information, 

668 see the Notes section below.) The argument `c` is an array of 

669 coefficients from low to high degree along each axis, e.g., [1,2,3] 

670 represents the series ``L_0 + 2*L_1 + 3*L_2`` while [[1,2],[1,2]] 

671 represents ``1*L_0(x)*L_0(y) + 1*L_1(x)*L_0(y) + 2*L_0(x)*L_1(y) + 

672 2*L_1(x)*L_1(y)`` if axis=0 is ``x`` and axis=1 is ``y``. 

673 

674 

675 Parameters 

676 ---------- 

677 c : array_like 

678 Array of Laguerre series coefficients. If `c` is multidimensional 

679 the different axis correspond to different variables with the 

680 degree in each axis given by the corresponding index. 

681 m : int, optional 

682 Order of integration, must be positive. (Default: 1) 

683 k : {[], list, scalar}, optional 

684 Integration constant(s). The value of the first integral at 

685 ``lbnd`` is the first value in the list, the value of the second 

686 integral at ``lbnd`` is the second value, etc. If ``k == []`` (the 

687 default), all constants are set to zero. If ``m == 1``, a single 

688 scalar can be given instead of a list. 

689 lbnd : scalar, optional 

690 The lower bound of the integral. (Default: 0) 

691 scl : scalar, optional 

692 Following each integration the result is *multiplied* by `scl` 

693 before the integration constant is added. (Default: 1) 

694 axis : int, optional 

695 Axis over which the integral is taken. (Default: 0). 

696 

697 .. versionadded:: 1.7.0 

698 

699 Returns 

700 ------- 

701 S : ndarray 

702 Laguerre series coefficients of the integral. 

703 

704 Raises 

705 ------ 

706 ValueError 

707 If ``m < 0``, ``len(k) > m``, ``np.ndim(lbnd) != 0``, or 

708 ``np.ndim(scl) != 0``. 

709 

710 See Also 

711 -------- 

712 lagder 

713 

714 Notes 

715 ----- 

716 Note that the result of each integration is *multiplied* by `scl`. 

717 Why is this important to note? Say one is making a linear change of 

718 variable :math:`u = ax + b` in an integral relative to `x`. Then 

719 :math:`dx = du/a`, so one will need to set `scl` equal to 

720 :math:`1/a` - perhaps not what one would have first thought. 

721 

722 Also note that, in general, the result of integrating a C-series needs 

723 to be "reprojected" onto the C-series basis set. Thus, typically, 

724 the result of this function is "unintuitive," albeit correct; see 

725 Examples section below. 

726 

727 Examples 

728 -------- 

729 >>> from numpy.polynomial.laguerre import lagint 

730 >>> lagint([1,2,3]) 

731 array([ 1., 1., 1., -3.]) 

732 >>> lagint([1,2,3], m=2) 

733 array([ 1., 0., 0., -4., 3.]) 

734 >>> lagint([1,2,3], k=1) 

735 array([ 2., 1., 1., -3.]) 

736 >>> lagint([1,2,3], lbnd=-1) 

737 array([11.5, 1. , 1. , -3. ]) 

738 >>> lagint([1,2], m=2, k=[1,2], lbnd=-1) 

739 array([ 11.16666667, -5. , -3. , 2. ]) # may vary 

740 

741 """ 

742 c = np.array(c, ndmin=1, copy=True) 

743 if c.dtype.char in '?bBhHiIlLqQpP': 

744 c = c.astype(np.double) 

745 if not np.iterable(k): 

746 k = [k] 

747 cnt = pu._deprecate_as_int(m, "the order of integration") 

748 iaxis = pu._deprecate_as_int(axis, "the axis") 

749 if cnt < 0: 

750 raise ValueError("The order of integration must be non-negative") 

751 if len(k) > cnt: 

752 raise ValueError("Too many integration constants") 

753 if np.ndim(lbnd) != 0: 

754 raise ValueError("lbnd must be a scalar.") 

755 if np.ndim(scl) != 0: 

756 raise ValueError("scl must be a scalar.") 

757 iaxis = normalize_axis_index(iaxis, c.ndim) 

758 

759 if cnt == 0: 

760 return c 

761 

762 c = np.moveaxis(c, iaxis, 0) 

763 k = list(k) + [0]*(cnt - len(k)) 

764 for i in range(cnt): 

765 n = len(c) 

766 c *= scl 

767 if n == 1 and np.all(c[0] == 0): 

768 c[0] += k[i] 

769 else: 

770 tmp = np.empty((n + 1,) + c.shape[1:], dtype=c.dtype) 

771 tmp[0] = c[0] 

772 tmp[1] = -c[0] 

773 for j in range(1, n): 

774 tmp[j] += c[j] 

775 tmp[j + 1] = -c[j] 

776 tmp[0] += k[i] - lagval(lbnd, tmp) 

777 c = tmp 

778 c = np.moveaxis(c, 0, iaxis) 

779 return c

782def lagval(x, c, tensor=True): 

783 """ 

784 Evaluate a Laguerre series at points x. 

785 

786 If `c` is of length `n + 1`, this function returns the value: 

787 

788 .. math:: p(x) = c_0 * L_0(x) + c_1 * L_1(x) + ... + c_n * L_n(x) 

789 

790 The parameter `x` is converted to an array only if it is a tuple or a 

791 list, otherwise it is treated as a scalar. In either case, either `x` 

792 or its elements must support multiplication and addition both with 

793 themselves and with the elements of `c`. 

794 

795 If `c` is a 1-D array, then `p(x)` will have the same shape as `x`. If 

796 `c` is multidimensional, then the shape of the result depends on the 

797 value of `tensor`. If `tensor` is true the shape will be c.shape[1:] + 

798 x.shape. If `tensor` is false the shape will be c.shape[1:]. Note that 

799 scalars have shape (,). 

800 

801 Trailing zeros in the coefficients will be used in the evaluation, so 

802 they should be avoided if efficiency is a concern. 

803 

804 Parameters 

805 ---------- 

806 x : array_like, compatible object 

807 If `x` is a list or tuple, it is converted to an ndarray, otherwise 

808 it is left unchanged and treated as a scalar. In either case, `x` 

809 or its elements must support addition and multiplication with 

810 with themselves and with the elements of `c`. 

811 c : array_like 

812 Array of coefficients ordered so that the coefficients for terms of 

813 degree n are contained in c[n]. If `c` is multidimensional the 

814 remaining indices enumerate multiple polynomials. In the two 

815 dimensional case the coefficients may be thought of as stored in 

816 the columns of `c`. 

817 tensor : boolean, optional 

818 If True, the shape of the coefficient array is extended with ones 

819 on the right, one for each dimension of `x`. Scalars have dimension 0 

820 for this action. The result is that every column of coefficients in 

821 `c` is evaluated for every element of `x`. If False, `x` is broadcast 

822 over the columns of `c` for the evaluation. This keyword is useful 

823 when `c` is multidimensional. The default value is True. 

824 

825 .. versionadded:: 1.7.0 

826 

827 Returns 

828 ------- 

829 values : ndarray, algebra_like 

830 The shape of the return value is described above. 

831 

832 See Also 

833 -------- 

834 lagval2d, laggrid2d, lagval3d, laggrid3d 

835 

836 Notes 

837 ----- 

838 The evaluation uses Clenshaw recursion, aka synthetic division. 

839 

840 Examples 

841 -------- 

842 >>> from numpy.polynomial.laguerre import lagval 

843 >>> coef = [1,2,3] 

844 >>> lagval(1, coef) 

845 -0.5 

846 >>> lagval([[1,2],[3,4]], coef) 

847 array([[-0.5, -4. ], 

848 [-4.5, -2. ]]) 

849 

850 """ 

851 c = np.array(c, ndmin=1, copy=False) 

852 if c.dtype.char in '?bBhHiIlLqQpP': 

853 c = c.astype(np.double) 

854 if isinstance(x, (tuple, list)): 

855 x = np.asarray(x) 

856 if isinstance(x, np.ndarray) and tensor: 

857 c = c.reshape(c.shape + (1,)*x.ndim) 

858 

859 if len(c) == 1: 

860 c0 = c[0] 

861 c1 = 0 

862 elif len(c) == 2: 

863 c0 = c[0] 

864 c1 = c[1] 

865 else: 

866 nd = len(c) 

867 c0 = c[-2] 

868 c1 = c[-1] 

869 for i in range(3, len(c) + 1): 

870 tmp = c0 

871 nd = nd - 1 

872 c0 = c[-i] - (c1*(nd - 1))/nd 

873 c1 = tmp + (c1*((2*nd - 1) - x))/nd 

874 return c0 + c1*(1 - x)

1086def lagvander(x, deg): 

1087 """Pseudo-Vandermonde matrix of given degree. 

1088 

1089 Returns the pseudo-Vandermonde matrix of degree `deg` and sample points 

1090 `x`. The pseudo-Vandermonde matrix is defined by 

1091 

1092 .. math:: V[..., i] = L_i(x) 

1093 

1094 where `0 <= i <= deg`. The leading indices of `V` index the elements of 

1095 `x` and the last index is the degree of the Laguerre polynomial. 

1096 

1097 If `c` is a 1-D array of coefficients of length `n + 1` and `V` is the 

1098 array ``V = lagvander(x, n)``, then ``np.dot(V, c)`` and 

1099 ``lagval(x, c)`` are the same up to roundoff. This equivalence is 

1100 useful both for least squares fitting and for the evaluation of a large 

1101 number of Laguerre series of the same degree and sample points. 

1102 

1103 Parameters 

1104 ---------- 

1105 x : array_like 

1106 Array of points. The dtype is converted to float64 or complex128 

1107 depending on whether any of the elements are complex. If `x` is 

1108 scalar it is converted to a 1-D array. 

1109 deg : int 

1110 Degree of the resulting matrix. 

1111 

1112 Returns 

1113 ------- 

1114 vander : ndarray 

1115 The pseudo-Vandermonde matrix. The shape of the returned matrix is 

1116 ``x.shape + (deg + 1,)``, where The last index is the degree of the 

1117 corresponding Laguerre polynomial. The dtype will be the same as 

1118 the converted `x`. 

1119 

1120 Examples 

1121 -------- 

1122 >>> from numpy.polynomial.laguerre import lagvander 

1123 >>> x = np.array([0, 1, 2]) 

1124 >>> lagvander(x, 3) 

1125 array([[ 1. , 1. , 1. , 1. ], 

1126 [ 1. , 0. , -0.5 , -0.66666667], 

1127 [ 1. , -1. , -1. , -0.33333333]]) 

1128 

1129 """ 

1130 ideg = pu._deprecate_as_int(deg, "deg") 

1131 if ideg < 0: 

1132 raise ValueError("deg must be non-negative") 

1133 

1134 x = np.array(x, copy=False, ndmin=1) + 0.0 

1135 dims = (ideg + 1,) + x.shape 

1136 dtyp = x.dtype 

1137 v = np.empty(dims, dtype=dtyp) 

1138 v[0] = x*0 + 1 

1139 if ideg > 0: 

1140 v[1] = 1 - x 

1141 for i in range(2, ideg + 1): 

1142 v[i] = (v[i-1]*(2*i - 1 - x) - v[i-2]*(i - 1))/i 

1143 return np.moveaxis(v, 0, -1)

1253def lagfit(x, y, deg, rcond=None, full=False, w=None): 

1254 """ 

1255 Least squares fit of Laguerre series to data. 

1256 

1257 Return the coefficients of a Laguerre series of degree `deg` that is the 

1258 least squares fit to the data values `y` given at points `x`. If `y` is 

1259 1-D the returned coefficients will also be 1-D. If `y` is 2-D multiple 

1260 fits are done, one for each column of `y`, and the resulting 

1261 coefficients are stored in the corresponding columns of a 2-D return. 

1262 The fitted polynomial(s) are in the form 

1263 

1264 .. math:: p(x) = c_0 + c_1 * L_1(x) + ... + c_n * L_n(x), 

1265 

1266 where `n` is `deg`. 

1267 

1268 Parameters 

1269 ---------- 

1270 x : array_like, shape (M,) 

1271 x-coordinates of the M sample points ``(x[i], y[i])``. 

1272 y : array_like, shape (M,) or (M, K) 

1273 y-coordinates of the sample points. Several data sets of sample 

1274 points sharing the same x-coordinates can be fitted at once by 

1275 passing in a 2D-array that contains one dataset per column. 

1276 deg : int or 1-D array_like 

1277 Degree(s) of the fitting polynomials. If `deg` is a single integer 

1278 all terms up to and including the `deg`'th term are included in the 

1279 fit. For NumPy versions >= 1.11.0 a list of integers specifying the 

1280 degrees of the terms to include may be used instead. 

1281 rcond : float, optional 

1282 Relative condition number of the fit. Singular values smaller than 

1283 this relative to the largest singular value will be ignored. The 

1284 default value is len(x)*eps, where eps is the relative precision of 

1285 the float type, about 2e-16 in most cases. 

1286 full : bool, optional 

1287 Switch determining nature of return value. When it is False (the 

1288 default) just the coefficients are returned, when True diagnostic 

1289 information from the singular value decomposition is also returned. 

1290 w : array_like, shape (`M`,), optional 

1291 Weights. If not None, the contribution of each point 

1292 ``(x[i],y[i])`` to the fit is weighted by `w[i]`. Ideally the 

1293 weights are chosen so that the errors of the products ``w[i]*y[i]`` 

1294 all have the same variance. The default value is None. 

1295 

1296 Returns 

1297 ------- 

1298 coef : ndarray, shape (M,) or (M, K) 

1299 Laguerre coefficients ordered from low to high. If `y` was 2-D, 

1300 the coefficients for the data in column k of `y` are in column 

1301 `k`. 

1302 

1303 [residuals, rank, singular_values, rcond] : list 

1304 These values are only returned if `full` = True 

1305 

1306 resid -- sum of squared residuals of the least squares fit 

1307 rank -- the numerical rank of the scaled Vandermonde matrix 

1308 sv -- singular values of the scaled Vandermonde matrix 

1309 rcond -- value of `rcond`. 

1310 

1311 For more details, see `linalg.lstsq`. 

1312 

1313 Warns 

1314 ----- 

1315 RankWarning 

1316 The rank of the coefficient matrix in the least-squares fit is 

1317 deficient. The warning is only raised if `full` = False. The 

1318 warnings can be turned off by 

1319 

1320 >>> import warnings 

1321 >>> warnings.simplefilter('ignore', np.RankWarning) 

1322 

1323 See Also 

1324 -------- 

1325 chebfit, legfit, polyfit, hermfit, hermefit 

1326 lagval : Evaluates a Laguerre series. 

1327 lagvander : pseudo Vandermonde matrix of Laguerre series. 

1328 lagweight : Laguerre weight function. 

1329 linalg.lstsq : Computes a least-squares fit from the matrix. 

1330 scipy.interpolate.UnivariateSpline : Computes spline fits. 

1331 

1332 Notes 

1333 ----- 

1334 The solution is the coefficients of the Laguerre series `p` that 

1335 minimizes the sum of the weighted squared errors 

1336 

1337 .. math:: E = \\sum_j w_j^2 * |y_j - p(x_j)|^2, 

1338 

1339 where the :math:`w_j` are the weights. This problem is solved by 

1340 setting up as the (typically) overdetermined matrix equation 

1341 

1342 .. math:: V(x) * c = w * y, 

1343 

1344 where `V` is the weighted pseudo Vandermonde matrix of `x`, `c` are the 

1345 coefficients to be solved for, `w` are the weights, and `y` are the 

1346 observed values. This equation is then solved using the singular value 

1347 decomposition of `V`. 

1348 

1349 If some of the singular values of `V` are so small that they are 

1350 neglected, then a `RankWarning` will be issued. This means that the 

1351 coefficient values may be poorly determined. Using a lower order fit 

1352 will usually get rid of the warning. The `rcond` parameter can also be 

1353 set to a value smaller than its default, but the resulting fit may be 

1354 spurious and have large contributions from roundoff error. 

1355 

1356 Fits using Laguerre series are probably most useful when the data can 

1357 be approximated by ``sqrt(w(x)) * p(x)``, where `w(x)` is the Laguerre 

1358 weight. In that case the weight ``sqrt(w(x[i])`` should be used 

1359 together with data values ``y[i]/sqrt(w(x[i])``. The weight function is 

1360 available as `lagweight`. 

1361 

1362 References 

1363 ---------- 

1364 .. [1] Wikipedia, "Curve fitting", 

1365 https://en.wikipedia.org/wiki/Curve_fitting 

1366 

1367 Examples 

1368 -------- 

1369 >>> from numpy.polynomial.laguerre import lagfit, lagval 

1370 >>> x = np.linspace(0, 10) 

1371 >>> err = np.random.randn(len(x))/10 

1372 >>> y = lagval(x, [1, 2, 3]) + err 

1373 >>> lagfit(x, y, 2) 

1374 array([ 0.96971004, 2.00193749, 3.00288744]) # may vary 

1375 

1376 """

1380def lagcompanion(c): 

1381 """ 

1382 Return the companion matrix of c. 

1383 

1384 The usual companion matrix of the Laguerre polynomials is already 

1385 symmetric when `c` is a basis Laguerre polynomial, so no scaling is 

1386 applied. 

1387 

1388 Parameters 

1389 ---------- 

1390 c : array_like 

1391 1-D array of Laguerre series coefficients ordered from low to high 

1392 degree. 

1393 

1394 Returns 

1395 ------- 

1396 mat : ndarray 

1397 Companion matrix of dimensions (deg, deg). 

1398 

1399 Notes 

1400 ----- 

1401 

1402 .. versionadded:: 1.7.0 

1403 

1404 """ 

1405 # c is a trimmed copy 

1406 [c] = pu.as_series([c]) 

1407 if len(c) < 2: 

1408 raise ValueError('Series must have maximum degree of at least 1.') 

1409 if len(c) == 2: 

1410 return np.array([[1 + c[0]/c[1]]]) 

1411 

1412 n = len(c) - 1 

1413 mat = np.zeros((n, n), dtype=c.dtype) 

1414 top = mat.reshape(-1)[1::n+1] 

1415 mid = mat.reshape(-1)[0::n+1] 

1416 bot = mat.reshape(-1)[n::n+1] 

1417 top[...] = -np.arange(1, n) 

1418 mid[...] = 2.*np.arange(n) + 1. 

1419 bot[...] = top 

1420 mat[:, -1] += (c[:-1]/c[-1])*n 

1421 return mat

1424def lagroots(c): 

1425 """ 

1426 Compute the roots of a Laguerre series. 

1427 

1428 Return the roots (a.k.a. "zeros") of the polynomial 

1429 

1430 .. math:: p(x) = \\sum_i c[i] * L_i(x). 

1431 

1432 Parameters 

1433 ---------- 

1434 c : 1-D array_like 

1435 1-D array of coefficients. 

1436 

1437 Returns 

1438 ------- 

1439 out : ndarray 

1440 Array of the roots of the series. If all the roots are real, 

1441 then `out` is also real, otherwise it is complex. 

1442 

1443 See Also 

1444 -------- 

1445 polyroots, legroots, chebroots, hermroots, hermeroots 

1446 

1447 Notes 

1448 ----- 

1449 The root estimates are obtained as the eigenvalues of the companion 

1450 matrix, Roots far from the origin of the complex plane may have large 

1451 errors due to the numerical instability of the series for such 

1452 values. Roots with multiplicity greater than 1 will also show larger 

1453 errors as the value of the series near such points is relatively 

1454 insensitive to errors in the roots. Isolated roots near the origin can 

1455 be improved by a few iterations of Newton's method. 

1456 

1457 The Laguerre series basis polynomials aren't powers of `x` so the 

1458 results of this function may seem unintuitive. 

1459 

1460 Examples 

1461 -------- 

1462 >>> from numpy.polynomial.laguerre import lagroots, lagfromroots 

1463 >>> coef = lagfromroots([0, 1, 2]) 

1464 >>> coef 

1465 array([ 2., -8., 12., -6.]) 

1466 >>> lagroots(coef) 

1467 array([-4.4408921e-16, 1.0000000e+00, 2.0000000e+00]) 

1468 

1469 """ 

1470 # c is a trimmed copy 

1471 [c] = pu.as_series([c]) 

1472 if len(c) <= 1: 

1473 return np.array([], dtype=c.dtype) 

1474 if len(c) == 2: 

1475 return np.array([1 + c[0]/c[1]]) 

1476 

1477 # rotated companion matrix reduces error 

1478 m = lagcompanion(c)[::-1,::-1] 

1479 r = la.eigvals(m) 

1480 r.sort() 

1481 return r

1484def laggauss(deg): 

1485 """ 

1486 Gauss-Laguerre quadrature. 

1487 

1488 Computes the sample points and weights for Gauss-Laguerre quadrature. 

1489 These sample points and weights will correctly integrate polynomials of 

1490 degree :math:`2*deg - 1` or less over the interval :math:`[0, \\inf]` 

1491 with the weight function :math:`f(x) = \\exp(-x)`. 

1492 

1493 Parameters 

1494 ---------- 

1495 deg : int 

1496 Number of sample points and weights. It must be >= 1. 

1497 

1498 Returns 

1499 ------- 

1500 x : ndarray 

1501 1-D ndarray containing the sample points. 

1502 y : ndarray 

1503 1-D ndarray containing the weights. 

1504 

1505 Notes 

1506 ----- 

1507 

1508 .. versionadded:: 1.7.0 

1509 

1510 The results have only been tested up to degree 100 higher degrees may 

1511 be problematic. The weights are determined by using the fact that 

1512 

1513 .. math:: w_k = c / (L'_n(x_k) * L_{n-1}(x_k)) 

1514 

1515 where :math:`c` is a constant independent of :math:`k` and :math:`x_k` 

1516 is the k'th root of :math:`L_n`, and then scaling the results to get 

1517 the right value when integrating 1. 

1518 

1519 """ 

1520 ideg = pu._deprecate_as_int(deg, "deg") 

1521 if ideg <= 0: 

1522 raise ValueError("deg must be a positive integer") 

1523 

1524 # first approximation of roots. We use the fact that the companion 

1525 # matrix is symmetric in this case in order to obtain better zeros. 

1526 c = np.array([0]*deg + [1]) 

1527 m = lagcompanion(c) 

1528 x = la.eigvalsh(m) 

1529 

1530 # improve roots by one application of Newton 

1531 dy = lagval(x, c) 

1532 df = lagval(x, lagder(c)) 

1533 x -= dy/df 

1534 

1535 # compute the weights. We scale the factor to avoid possible numerical 

1536 # overflow. 

1537 fm = lagval(x, c[1:]) 

1538 fm /= np.abs(fm).max() 

1539 df /= np.abs(df).max() 

1540 w = 1/(fm * df) 

1541 

1542 # scale w to get the right value, 1 in this case 

1543 w /= w.sum() 

1544 

1545 return x, w
