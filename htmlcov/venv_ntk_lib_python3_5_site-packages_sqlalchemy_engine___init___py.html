8"""SQL connections, SQL execution and high-level DB-API interface. 

9 

10The engine package defines the basic components used to interface 

11DB-API modules with higher-level statement construction, 

12connection-management, execution and result contexts. The primary 

13"entry point" class into this package is the Engine and its public 

14constructor ``create_engine()``. 

15 

16This package includes: 

17 

18base.py 

19 Defines interface classes and some implementation classes which 

20 comprise the basic components used to interface between a DB-API, 

21 constructed and plain-text statements, connections, transactions, 

22 and results. 

23 

24default.py 

25 Contains default implementations of some of the components defined 

26 in base.py. All current database dialects use the classes in 

27 default.py as base classes for their own database-specific 

28 implementations. 

29 

30strategies.py 

31 The mechanics of constructing ``Engine`` objects are represented 

32 here. Defines the ``EngineStrategy`` class which represents how 

33 to go from arguments specified to the ``create_engine()`` 

34 function, to a fully constructed ``Engine``, including 

35 initialization of connection pooling, dialects, and specific 

36 subclasses of ``Engine``. 

37 

38threadlocal.py 

39 The ``TLEngine`` class is defined here, which is a subclass of 

40 the generic ``Engine`` and tracks ``Connection`` and 

41 ``Transaction`` objects against the identity of the current 

42 thread. This allows certain programming patterns based around 

43 the concept of a "thread-local connection" to be possible. 

44 The ``TLEngine`` is created by using the "threadlocal" engine 

45 strategy in conjunction with the ``create_engine()`` function. 

46 

47url.py 

48 Defines the ``URL`` class which represents the individual 

49 components of a string URL passed to ``create_engine()``. Also 

50 defines a basic module-loading strategy for the dialect specifier 

51 within a URL. 

52"""

85def create_engine(*args, **kwargs): 

86 """Create a new :class:`.Engine` instance. 

87 

88 The standard calling form is to send the URL as the 

89 first positional argument, usually a string 

90 that indicates database dialect and connection arguments:: 

91 

92 

93 engine = create_engine("postgresql://scott:tiger@localhost/test") 

94 

95 Additional keyword arguments may then follow it which 

96 establish various options on the resulting :class:`.Engine` 

97 and its underlying :class:`.Dialect` and :class:`.Pool` 

98 constructs:: 

99 

100 engine = create_engine("mysql://scott:tiger@hostname/dbname", 

101 encoding='latin1', echo=True) 

102 

103 The string form of the URL is 

104 ``dialect[+driver]://user:password@host/dbname[?key=value..]``, where 

105 ``dialect`` is a database name such as ``mysql``, ``oracle``, 

106 ``postgresql``, etc., and ``driver`` the name of a DBAPI, such as 

107 ``psycopg2``, ``pyodbc``, ``cx_oracle``, etc. Alternatively, 

108 the URL can be an instance of :class:`~sqlalchemy.engine.url.URL`. 

109 

110 ``**kwargs`` takes a wide variety of options which are routed 

111 towards their appropriate components. Arguments may be specific to 

112 the :class:`.Engine`, the underlying :class:`.Dialect`, as well as the 

113 :class:`.Pool`. Specific dialects also accept keyword arguments that 

114 are unique to that dialect. Here, we describe the parameters 

115 that are common to most :func:`.create_engine()` usage. 

116 

117 Once established, the newly resulting :class:`.Engine` will 

118 request a connection from the underlying :class:`.Pool` once 

119 :meth:`.Engine.connect` is called, or a method which depends on it 

120 such as :meth:`.Engine.execute` is invoked. The :class:`.Pool` in turn 

121 will establish the first actual DBAPI connection when this request 

122 is received. The :func:`.create_engine` call itself does **not** 

123 establish any actual DBAPI connections directly. 

124 

125 .. seealso:: 

126 

127 :doc:`/core/engines`

129 :doc:`/dialects/index`

133 :param case_sensitive=True: if False, result column names 

134 will match in a case-insensitive fashion, that is, 

135 ``row['SomeColumn']``. 

136 

137 :param connect_args: a dictionary of options which will be 

138 passed directly to the DBAPI's ``connect()`` method as 

139 additional keyword arguments. See the example 

140 at :ref:`custom_dbapi_args`.

142 :param convert_unicode=False: if set to True, causes 

143 all :class:`.String` datatypes to act as though the 

144 :paramref:`.String.convert_unicode` flag has been set to ``True``, 

145 regardless of a setting of ``False`` on an individual :class:`.String` 

146 type. This has the effect of causing all :class:`.String` -based 

147 columns to accommodate Python Unicode objects directly as though the 

148 datatype were the :class:`.Unicode` type. 

149

154 All modern DBAPIs now support Python Unicode directly and this

166 string ``"debug"``, result rows will be printed to the standard output 

167 as well. The ``echo`` attribute of ``Engine`` can be modified at any 

168 time to turn logging on and off; direct control of logging is also 

169 available using the standard Python ``logging`` module.

190 :param empty_in_strategy: The SQL compilation strategy to use when 

191 rendering an IN or NOT IN expression for :meth:`.ColumnOperators.in_` 

192 where the right-hand side 

193 is an empty set. This is a string value that may be one of 

194 ``static``, ``dynamic``, or ``dynamic_warn``. The ``static`` 

195 strategy is the default, and an IN comparison to an empty set 

196 will generate a simple false expression "1 != 1". The ``dynamic`` 

197 strategy behaves like that of SQLAlchemy 1.1 and earlier, emitting 

198 a false expression of the form "expr != expr", which has the effect 

199 of evaluting to NULL in the case of a null expression. 

200 ``dynamic_warn`` is the same as ``dynamic``, however also emits a 

201 warning when an empty set is encountered; this because the "dynamic" 

202 comparison is typically poorly performing on most databases.

211 the DBAPI.** Most modern DBAPIs feature some degree of

213 what you see in Python 2 as a string of the form

218 for those cases where the DBAPI handles unicode 

219 directly.

223 configured to handle unicode to the greatest

242 * the string column names retrieved from the DBAPI's 

243 ``cursor.description`` attribute. 

244

247 which in Python 3 are just known as ``str``. In Python 2, 

248 the DBAPI does not specify unicode behavior at all,

482def engine_from_config(configuration, prefix="sqlalchemy.", **kwargs): 

483 """Create a new Engine instance using a configuration dictionary. 

484 

485 The dictionary is typically produced from a config file. 

486 

487 The keys of interest to ``engine_from_config()`` should be prefixed, e.g. 

488 ``sqlalchemy.url``, ``sqlalchemy.echo``, etc. The 'prefix' argument 

489 indicates the prefix to be searched for. Each matching key (after the 

490 prefix is stripped) is treated as though it were the corresponding keyword 

491 argument to a :func:`.create_engine` call. 

492 

493 The only required key is (assuming the default prefix) ``sqlalchemy.url``, 

494 which provides the :ref:`database URL <database_urls>`. 

495 

496 A select set of keyword arguments will be "coerced" to their 

497 expected type based on string values. The set of arguments 

498 is extensible per-dialect using the ``engine_config_types`` accessor. 

499 

500 :param configuration: A dictionary (typically produced from a config file, 

501 but this is not a requirement). Items whose keys start with the value 

502 of 'prefix' will have that prefix stripped, and will then be passed to 

503 :ref:`create_engine`. 

504 

505 :param prefix: Prefix to match and then strip from keys 

506 in 'configuration'. 

507 

508 :param kwargs: Each keyword argument to ``engine_from_config()`` itself 

509 overrides the corresponding item taken from the 'configuration' 

510 dictionary. Keyword arguments should *not* be prefixed. 

511 

512 """ 

513 

514 options = dict( 

515 (key[len(prefix) :], configuration[key]) 

516 for key in configuration 

517 if key.startswith(prefix) 

518 ) 

519 options["_coerce_config"] = True 

520 options.update(kwargs) 

521 url = options.pop("url") 

522 return create_engine(url, **options)
