7''' Provide property types for Bokeh models 

8 

9Properties are objects that can be assigned as class attributes on Bokeh 

10models, to provide automatic serialization, validation, and documentation. 

11 

12This documentation is broken down into the following sections: 

13 

14.. contents:: 

15 :local: 

16 

17Overview 

18-------- 

19 

20There are many property types defined in the module, for example ``Int`` to 

21represent integral values, ``Seq`` to represent sequences (e.g. lists or 

22tuples, etc.). Properties can also be combined: ``Seq(Float)`` represents 

23a sequence of floating point values. 

24 

25For example, the following defines a model that has integer, string, and 

26list[float] properties: 

27 

28.. code-block:: python 

29 

30 class SomeModel(Model): 

31 foo = Int 

32 bar = String(default="something") 

33 baz = List(Float, help="docs for baz prop") 

34 

35As seen, properties can be declared as just the property type, e.g. 

36``foo = Int``, in which case the properties are automatically instantiated 

37on new Model objects. Or the property can be instantiated on the class, 

38and configured with default values and help strings. 

39 

40The properties of this class can be initialized by specifying keyword 

41arguments to the initializer: 

42 

43.. code-block:: python 

44 

45 m = SomeModel(foo=10, bar="a str", baz=[1,2,3,4]) 

46 

47But also by setting the attributes on an instance: 

48 

49.. code-block:: python 

50 

51 m.foo = 20 

52 

53Attempts to set a property to a value of the wrong type will 

54result in a ``ValueError`` exception: 

55 

56.. code-block:: python 

57 

58 >>> m.foo = 2.3 

59 Traceback (most recent call last): 

60 

61 << traceback omitted >> 

62 

63 ValueError: expected a value of type Integral, got 2.3 of type float 

64 

65Models with properties know how to serialize themselves, to be understood 

66by BokehJS. Additionally, any help strings provided on properties can be 

67easily and automatically extracted with the Sphinx extensions in the 

68:ref:`bokeh.sphinxext` module. 

69

151Validation Control 

152------------------ 

153 

154By default, Bokeh properties perform type validation on values. This helps to 

155ensure the consistency of any data exchanged between Python and JavaScript, as 

156well as provide detailed and immediate feedback to users if they attempt to 

157set values of the wrong type. However, these type checks incur some overhead. 

158In some cases it may be desirable to turn off validation in specific places, 

159or even entirely, in order to boost performance. The following API is available 

160to control when type validation occurs. 

161 

162.. autoclass:: validate 

163.. autofunction:: without_property_validation
