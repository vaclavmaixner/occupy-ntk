15 """ 

16 Create an array. 

17 

18 .. versionadded:: 0.24.0 

19 

20 Parameters 

21 ---------- 

22 data : Sequence of objects 

23 The scalars inside `data` should be instances of the 

24 scalar type for `dtype`. It's expected that `data` 

25 represents a 1-dimensional array of data. 

26 

27 When `data` is an Index or Series, the underlying array 

28 will be extracted from `data`. 

29 

30 dtype : str, np.dtype, or ExtensionDtype, optional 

31 The dtype to use for the array. This may be a NumPy 

32 dtype or an extension type registered with pandas using 

33 :meth:`pandas.api.extensions.register_extension_dtype`. 

34 

35 If not specified, there are two possibilities: 

36 

37 1. When `data` is a :class:`Series`, :class:`Index`, or 

38 :class:`ExtensionArray`, the `dtype` will be taken 

39 from the data. 

40 2. Otherwise, pandas will attempt to infer the `dtype` 

41 from the data. 

42 

43 Note that when `data` is a NumPy array, ``data.dtype`` is 

44 *not* used for inferring the array type. This is because 

45 NumPy cannot represent all the types of data that can be 

46 held in extension arrays. 

47 

48 Currently, pandas will infer an extension dtype for sequences of 

49 

50 ============================== ===================================== 

51 Scalar Type Array Type 

52 ============================== ===================================== 

53 :class:`pandas.Interval` :class:`pandas.arrays.IntervalArray` 

54 :class:`pandas.Period` :class:`pandas.arrays.PeriodArray` 

55 :class:`datetime.datetime` :class:`pandas.arrays.DatetimeArray` 

56 :class:`datetime.timedelta` :class:`pandas.arrays.TimedeltaArray` 

57 ============================== ===================================== 

58 

59 For all other cases, NumPy's usual inference rules will be used. 

60 

61 copy : bool, default True 

62 Whether to copy the data, even if not necessary. Depending 

63 on the type of `data`, creating the new array may require 

64 copying data, even if ``copy=False``. 

65 

66 Returns 

67 ------- 

68 ExtensionArray 

69 The newly created array. 

70 

71 Raises 

72 ------ 

73 ValueError 

74 When `data` is not 1-dimensional. 

75 

76 See Also 

77 -------- 

78 numpy.array : Construct a NumPy array. 

79 Series : Construct a pandas Series. 

80 Index : Construct a pandas Index. 

81 arrays.PandasArray : ExtensionArray wrapping a NumPy array. 

82 Series.array : Extract the array stored within a Series. 

83 

84 Notes 

85 ----- 

86 Omitting the `dtype` argument means pandas will attempt to infer the 

87 best array type from the values in the data. As new array types are 

88 added by pandas and 3rd party libraries, the "best" array type may 

89 change. We recommend specifying `dtype` to ensure that 

90 

91 1. the correct array type for the data is returned 

92 2. the returned array type doesn't change as new extension types 

93 are added by pandas and third-party libraries 

94 

95 Additionally, if the underlying memory representation of the returned 

96 array matters, we recommend specifying the `dtype` as a concrete object 

97 rather than a string alias or allowing it to be inferred. For example, 

98 a future version of pandas or a 3rd-party library may include a 

99 dedicated ExtensionArray for string data. In this event, the following 

100 would no longer return a :class:`arrays.PandasArray` backed by a NumPy 

101 array. 

102 

103 >>> pd.array(['a', 'b'], dtype=str) 

104 <PandasArray> 

105 ['a', 'b'] 

106 Length: 2, dtype: str32 

107 

108 This would instead return the new ExtensionArray dedicated for string 

109 data. If you really need the new array to be backed by a NumPy array, 

110 specify that in the dtype. 

111 

112 >>> pd.array(['a', 'b'], dtype=np.dtype("<U1")) 

113 <PandasArray> 

114 ['a', 'b'] 

115 Length: 2, dtype: str32 

116 

117 Or use the dedicated constructor for the array you're expecting, and 

118 wrap that in a PandasArray 

119 

120 >>> pd.array(np.array(['a', 'b'], dtype='<U1')) 

121 <PandasArray> 

122 ['a', 'b'] 

123 Length: 2, dtype: str32 

124 

125 Finally, Pandas has arrays that mostly overlap with NumPy 

126 

127 * :class:`arrays.DatetimeArray` 

128 * :class:`arrays.TimedeltaArray` 

129 

130 When data with a ``datetime64[ns]`` or ``timedelta64[ns]`` dtype is 

131 passed, pandas will always return a ``DatetimeArray`` or ``TimedeltaArray`` 

132 rather than a ``PandasArray``. This is for symmetry with the case of 

133 timezone-aware data, which NumPy does not natively support. 

134 

135 >>> pd.array(['2015', '2016'], dtype='datetime64[ns]') 

136 <DatetimeArray> 

137 ['2015-01-01 00:00:00', '2016-01-01 00:00:00'] 

138 Length: 2, dtype: datetime64[ns] 

139 

140 >>> pd.array(["1H", "2H"], dtype='timedelta64[ns]') 

141 <TimedeltaArray> 

142 ['01:00:00', '02:00:00'] 

143 Length: 2, dtype: timedelta64[ns] 

144 

145 Examples 

146 -------- 

147 If a dtype is not specified, `data` is passed through to 

148 :meth:`numpy.array`, and a :class:`arrays.PandasArray` is returned. 

149 

150 >>> pd.array([1, 2]) 

151 <PandasArray> 

152 [1, 2] 

153 Length: 2, dtype: int64 

154 

155 Or the NumPy dtype can be specified 

156 

157 >>> pd.array([1, 2], dtype=np.dtype("int32")) 

158 <PandasArray> 

159 [1, 2] 

160 Length: 2, dtype: int32 

161 

162 You can use the string alias for `dtype` 

163 

164 >>> pd.array(['a', 'b', 'a'], dtype='category') 

165 [a, b, a] 

166 Categories (2, object): [a, b] 

167 

168 Or specify the actual dtype 

169 

170 >>> pd.array(['a', 'b', 'a'], 

171 ... dtype=pd.CategoricalDtype(['a', 'b', 'c'], ordered=True)) 

172 [a, b, a] 

173 Categories (3, object): [a < b < c] 

174 

175 Because omitting the `dtype` passes the data through to NumPy, 

176 a mixture of valid integers and NA will return a floating-point 

177 NumPy array. 

178 

179 >>> pd.array([1, 2, np.nan]) 

180 <PandasArray> 

181 [1.0, 2.0, nan] 

182 Length: 3, dtype: float64 

183 

184 To use pandas' nullable :class:`pandas.arrays.IntegerArray`, specify 

185 the dtype: 

186 

187 >>> pd.array([1, 2, np.nan], dtype='Int64') 

188 <IntegerArray> 

189 [1, 2, NaN] 

190 Length: 3, dtype: Int64 

191 

192 Pandas will infer an ExtensionArray for some types of data: 

193 

194 >>> pd.array([pd.Period('2000', freq="D"), pd.Period("2000", freq="D")]) 

195 <PeriodArray> 

196 ['2000-01-01', '2000-01-01'] 

197 Length: 2, dtype: period[D] 

198 

199 `data` must be 1-dimensional. A ValueError is raised when the input 

200 has the wrong dimensionality. 

201 

202 >>> pd.array(1) 

203 Traceback (most recent call last): 

204 ... 

205 ValueError: Cannot pass scalar '1' to 'pandas.array'. 

206 """
