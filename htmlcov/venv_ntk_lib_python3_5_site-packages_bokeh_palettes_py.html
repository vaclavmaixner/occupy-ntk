104""" Provide a collection of palettes for color mapping. 

105 

106In the context of Bokeh, a *palette* is a simple plain Python list of (hex) RGB color 

107strings. For example the ``Blues8`` palette which looks like :bokeh-palette:`Blues8` 

108is defined as: 

109 

110.. code-block:: python 

111 

112 ['#084594', '#2171b5', '#4292c6', '#6baed6', '#9ecae1', '#c6dbef', '#deebf7', '#f7fbff'] 

113 

114This module contains the following sets of palettes: 

115 

116* All `ColorBrewer`_ palettes 

117* Categorical `D3`_ palettes 

118* The `Matplotlib`_ palettes Magma, Inferno, Plasma, and Viridis 

119* Palettes designed for color-deficient usability 

120 

121Additionally, you can also use any of the 256-color perceptually uniform 

122Bokeh palettes from the external `colorcet`_ package, if it is installed. 

123 

124---- 

125 

126Every pre-built palette is available as a module attributes, e.g. 

127``bokeh.palettes.YlGn3`` or ``bokeh.palettes.Viridis256``. The name of each 

128pre-built palette can be found in the ``__palettes__`` module attribute. 

129 

130There are also functions such as :func:`~bokeh.palettes.magma` and 

131:func:`~bokeh.palettes.viridis` that can generate lists of colors of 

132arbitrary size from special larger palettes. 

133 

134The Brewer palettes are also collected and grouped by name in a 

135``brewer`` dictionary, e.g.: ``brewer['Spectral'][6]``. Similarly there are 

136attributes ``d3``, ``mpl``, and ``colorblind`` that have dictionaries 

137corresponding to the those groups of palettes. 

138 

139Finally, all palettes are collected in the ``all_palettes`` palettes 

140module attribute, and the "small" palettes (i.e. excluding the ones with 256 

141colors) are collected and in a ``small_palettes`` attribute. 

142

203----------------

206following notable attributes in the ``bokeh.palettes`` module: 

207 

208.. data:: __palettes__ 

209 

210 An alphabetical list of the names of all individual palettes in this 

211 module. 

212

364# In order to prevent users from unintentionally modifying built-in palettes 

365# (which can result in subtle bugs), the bokeh.palettes modules uses the 

366# "class module" trick, to install a _PalettesModule instance in place of the 

367# actual module. The _PalettesModule should provide palettes as simple methods 

368# accepting only self, that return *new* lists of colors. These functions will 

369# automatically be turned into property getters with the _autoprop class 

370# decorator below, making the palettes appear to be simple module attributes, 

371# but guaranteeing that every usage is a new copy of the list of colors. 

372# 

373# An example of an appropriate palette method would be: 

374# 

375# def YlGn3(self): return ["#31a354", "#addd8e", "#f7fcb9"] 

376# 

377# Once defined, this will be available to users as bokeh.palettes.YlGn3 

378# 

379# Any "real" functions that the "module" should have should be added to the 

380# exclusion list inside _autoprop, or have their names start with "_". Any 

381# other methods on _PalettesModule will be automatically made into properties. 

382# 

383# Because of the complicated and unusual nature of the "class module", Sphinx 

384# requires some manual assistance to correctly generate documentation. When 

385# making any changes to this file, please makes sure to check and appropriately 

386# update the docstring at the top.

400def _autoprop(cls): 

401 for k, v in cls.__dict__.items(): 

402 if k.startswith('_'): continue 

403 if k in ['linear_palette', 'diverging_palette', 'magma', 'inferno', 'plasma', 'viridis', 'cividis', 'grey', 'gray', 'turbo']: 

404 continue 

405 setattr(cls, k, property(v)) 

406 return cls

1315 def linear_palette(self, palette, n): 

1316 ''' Generate a new palette as a subset of a given palette. 

1317 

1318 Given an input ``palette``, take ``n`` colors from it by dividing its 

1319 length into ``n`` (approximately) evenly spaced indices. 

1320 

1321 Args: 

1322 

1323 palette (list[str]) : a list of hex RGB color strings 

1324 n (int) : the size of the output palette to generate 

1325 

1326 Returns: 

1327 list [str] : a list of hex RGB color strings 

1328 

1329 Raises: 

1330 ``ValueError`` if ``n > len(palette)`` 

1331 

1332 ''' 

1333 import math 

1334 import numpy as np 

1335 if n > len(palette): 

1336 raise ValueError("Requested %(r)s colors, function can only return colors up to the base palette's length (%(l)s)" % dict(r=n, l=len(palette))) 

1337 return [ palette[int(math.floor(i))] for i in np.linspace(0, len(palette)-1, num=n) ]

1339 def diverging_palette(self, palette1, palette2, n, midpoint=0.5): 

1340 ''' Generate a new palette by combining exactly two input palettes. 

1341 

1342 Given an input ``palette1`` and ``palette2``, take a combined ``n`` colors, 

1343 and combine input palettes at the relative ``midpoint``. 

1344 ``palette1`` and ``palette2`` are meant to be sequential palettes that proceed 

1345 left to right from perceptually dark to light colors. In that case the returned 

1346 palette is comprised of the input palettes connected at perceptually light ends. 

1347 Palettes are combined by piecewise linear interpolation. 

1348 

1349 Args: 

1350 

1351 palette1 (list[str]) : 

1352 A list of hex RGB color strings for the first palette 

1353 

1354 palette2 (list[str]) : 

1355 A list of hex RGB color strings for the second palette 

1356 

1357 n (int) : 

1358 The size of the output palette to generate 

1359 

1360 midpoint (float, optional) : 

1361 Relative position in the returned palette where input palettes are 

1362 connected (default: 0.5) 

1363 

1364 Returns: 

1365 list [str] : a list of hex RGB color strings 

1366 

1367 Raises: 

1368 ``ValueError`` if n is greater than the possible combined length the input palettes 

1369 ''' 

1370 

1371 # flip palette2 so that perceptually light colors are joined 

1372 palette2 = palette2[::-1] 

1373 

1374 # determine number of colors from each palette 

1375 n1 = int(round(midpoint * n)) 

1376 n2 = int(round((1 - midpoint) * n)) 

1377 

1378 # return piecewise linear interpolation of colors 

1379 return self.linear_palette(palette1, n1) + self.linear_palette(palette2, n2)
