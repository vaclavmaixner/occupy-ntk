636def result_type(*arrays_and_dtypes): 

637 """ 

638 result_type(*arrays_and_dtypes) 

639 

640 Returns the type that results from applying the NumPy 

641 type promotion rules to the arguments. 

642 

643 Type promotion in NumPy works similarly to the rules in languages 

644 like C++, with some slight differences. When both scalars and 

645 arrays are used, the array's type takes precedence and the actual value 

646 of the scalar is taken into account. 

647 

648 For example, calculating 3*a, where a is an array of 32-bit floats, 

649 intuitively should result in a 32-bit float output. If the 3 is a 

650 32-bit integer, the NumPy rules indicate it can't convert losslessly 

651 into a 32-bit float, so a 64-bit float should be the result type. 

652 By examining the value of the constant, '3', we see that it fits in 

653 an 8-bit integer, which can be cast losslessly into the 32-bit float.
