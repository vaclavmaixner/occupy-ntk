7''' Control global configuration options with environment variables. 

8A global settings object that other parts of Bokeh can refer to. 

9 

10Defined Settings 

11~~~~~~~~~~~~~~~~ 

12 

13Settings are accessible on the ``bokeh.settings.settings`` instance, via 

14accessor methods. For instance: 

15 

16.. code-block:: python 

17 

18 settings.minified() 

19 

20Bokeh provides the following defined settings: 

21 

22.. bokeh-settings:: settings 

23 :module: bokeh.settings 

24 

25Precedence 

26~~~~~~~~~~ 

27 

28Setting values are always looked up in the following prescribed order: 

29 

30immediately supplied values 

31 These are values that are passed to the setting: 

32 

33 .. code-block:: python 

34 

35 settings.minified(minified_val) 

36 

37 If ``minified_val`` is not None, then it will be returned, as-is. 

38 Otherwise, if None is passed, then the setting will continute to look 

39 down the search order for a value. This is useful for passing optional 

40 function paramters that are None by default. If the parameter is passed 

41 to the function, then it will take precedence. 

42 

43previously user-set values 

44 If the value is set explicity in code: 

45 

46 .. code-block:: python 

47 

48 settings.minified = False 

49 

50 Then this value will take precedence over other sources. Applications 

51 may use this ability to set values supplied on the command line, so that 

52 they take precedence over environment variables. 

53 

54user-specified config override file 

55 Values from a YAML configuration file that is explicitly loaded: 

56 

57 .. code-block:: python 

58 

59 settings.load_config("/path/to/bokeh.yaml) 

60 

61 Any values from ``bokeh.yaml`` will take precedence over the sources 

62 below. Applications may offer command line arguments to load such a 

63 file. e.g. ``bokeh serve --use-config myconf.yaml`` 

64 

65environment variable 

66 Values found in the associated environment variables: 

67 

68 .. code-block:: sh 

69 

70 BOKEH_MINIFIED=no bokeh serve app.py 

71 

72local user config file 

73 Bokeh will look for a YAML configuration file in the current user's 

74 home directory ``${HOME}/.bokeh/bokeh.yaml``. 

75 

76global system configuration (not yet implemented) 

77 Future support is planned to load Bokeh settings from global system 

78 configurations. 

79 

80implicit defaults 

81 These are default values defined by the setting declarations. There are 

82 regular defaults, as well as "dev" defaults that are used when the 

83 environment variable ``BOKEH_DEV=yes`` is set. 

84 

85If no value is obtained after searching all of these locations, then a 

86RuntimeError will be raised. 

87

142def convert_bool(value): 

143 ''' Convert a string to True or False 

144 

145 If a boolean is passed in, it is returned as-is. Otherwise the function 

146 maps the following strings, ignoring case: 

147 

148 * "yes", "1", "on" -> True 

149 * "no", "0", "off" -> False 

150 

151 Args: 

152 value (str): 

153 A string value to convert to bool 

154 

155 Returns: 

156 bool 

157 

158 Raises: 

159 ValueError 

160 

161 ''' 

162 if value in (True, False): 

163 return value 

164 

165 val = value.lower() 

166 if val in ["yes", "1", "on", "true", "True"]: 

167 return True 

168 if val in ["no", "0", "off", "false", "False"]: 

169 return False 

170 

171 raise ValueError("Cannot convert {} to boolean value".format(value))

204def convert_logging(value): 

205 '''Convert a string to a Python logging level 

206 

207 If a log level is passed in, it is returned as-is. Otherwise the function 

208 understands the following strings, ignoring case: 

209 

210 * "critical" 

211 * "error" 

212 * "warning"

217 

218 Args:

220 A string value to convert to a logging level

243class PrioritizedSetting(object): 

244 ''' Return a value for a global setting according to configuration precedence. 

245 

246 The following methods are searched in order for the setting: 

247 

248 6. immediately supplied values 

249 5. previously user-set values (e.g. set from command line) 

250 4. user-specified config override file 

251 3. environment variable 

252 2. local user config file 

253 1. global system config file (not yet implemented) 

254 0. implicit defaults 

255 

256 Ref: https://stackoverflow.com/a/11077282/3406693 

257 

258 If a value cannot be determined, a ValueError is raised. 

259 

260 The ``env_var`` argument specifies the name of an environment to check for 

261 setting values, e.g. ``"BOKEH_LOG_LEVEL"``. 

262 

263 The optional ``default`` argument specified an implicit default value for 

264 the setting that is returned if no other methods provide a value. 

265 

266 A ``convert`` agument may be provided to convert values before they are 

267 returned. For instance to concert log levels in environment variables 

268 to ``logging`` module values. 

269 ''' 

270 

271 def __init__(self, name, env_var=None, default=_Unset, dev_default=_Unset, convert=None, help=""): 

272 self._convert = convert if convert else convert_str 

273 self._default = default 

274 self._dev_default = dev_default 

275 self._env_var = env_var 

276 self._help = help 

277 self._name = name 

278 self._parent = None 

279 self._user_value = _Unset 

280 

281 def __call__(self, value=None): 

282 '''Return the setting value according to the standard precedence. 

283 

284 Args: 

285 value (any, optional): 

286 An optional immediate value. If not None, the value will 

287 be converted, then returned. 

288 

289 Returns: 

290 str or int or float 

291 

292 Raises: 

293 RuntimeError 

294 ''' 

295 

296 # 6. immediate values 

297 if value is not None: 

298 return self._convert(value) 

299 

300 # 5. previously user-set value 

301 if self._user_value is not _Unset: 

302 return self._convert(self._user_value) 

303 

304 # 4. user-named config file 

305 if self._parent and self._name in self._parent.config_override: 

306 return self._convert(self._parent.config_override[self._name]) 

307 

308 # 3. environment variable 

309 if self._env_var and self._env_var in os.environ: 

310 return self._convert(os.environ[self._env_var]) 

311 

312 # 2. local config file 

313 if self._parent and self._name in self._parent.config_user: 

314 return self._convert(self._parent.config_user[self._name]) 

315 

316 # 1. global config file 

317 if self._parent and self._name in self._parent.config_system: 

318 return self._convert(self._parent.config_system[self._name]) 

319 

320 # 0. implicit defaults 

321 if is_dev() and self._dev_default is not _Unset: 

322 return self._convert(self._dev_default) 

323 if self._default is not _Unset: 

324 return self._convert(self._default) 

325 

326 raise RuntimeError("No configured value found for setting %r" % self._name)
